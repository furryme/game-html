# 重力翻转

**一句话：** 消行后获得"翻转权"，主动选择重力方向（上下左右四选一），方块从新方向掉落。

| 维度 | 评分 |
|------|------|
| 好玩度 | ⭐⭐⭐⭐⭐ 彻底颠覆"从上往下"的肌肉记忆，每次翻转都是全局决策 |
| 传播力 | ⭐⭐⭐⭐⭐ "那个重力会变的俄罗斯方块"，录屏传播效果极佳 |
| 新意度 | ⭐⭐⭐⭐⭐ 将 Tetris 从"垂直堆叠游戏"变成"空间方向游戏" |
| 实现难度 | ⭐⭐⭐ 中，Canvas 2D 坐标系旋转 + 渐进式开放 |

## 核心机制详解

### 基础规则
- 标准 7-bag 随机方块，10×20 棋盘
- 标准消行规则，但消行后不直接进入下一块，而是**获得翻转权**
- 单消获得 1 次翻转权，双消获得 2 次，以此类推
- 翻转权可累积，上限 3 次（防止贪分导致的策略失衡）

### 翻转系统
- 按下 R 键（或点击翻转按钮）消耗 1 次翻转权
- 翻转后，重力方向改变：上/下/左/右四选一，循环切换
- 当前下落中的方块会立即从新重力方向重新定位到"入口"位置
- 已锁定的方块保持原位不变，只是后续方块从新方向下落
- **棋盘显示**：四边各有一个重力入口指示器，当前方向高亮显示

### 渐进式难度
| 阶段 | 开放角度 | 触发条件 |
|------|---------|---------|
| Phase 1 | 仅 180°（上/下） | 游戏开始 |
| Phase 2 | 四向全开 | 达到 5 级 |
| Phase 3 | 翻转后附带攻击 | 达到 10 级 |

### 对战模式（共享重力）
- 玩家翻转重力时，**对手棋盘也同时翻转**
- "我翻转是为了自己消行，但对手也被连累"的对抗博弈
- 对手收到的翻转方向固定为"下"，增加对手操作压力
- 翻转变化的视觉特效同步显示在对手棋盘边缘

## 成瘾循环

获得翻转权 → "翻还是不翻"的策略抉择 → 翻转后空间感知重建的挑战 → 新角度下消行爽感 → 更多翻转权 → 循环

核心张力：**翻转的收益（在新角度消行）vs 翻转的成本（旧角度的结构在新重力下可能崩塌）**

## "啊哈"时刻

1. **第一次翻转**：棋盘瞬间倒转，方块"从右边"掉落——大脑需要 1-2 秒重新建立空间感知
2. **共享重力对战**：你翻转重力消了 4 行，对手的棋盘同时翻转导致大量方块坠落，直接 Game Over
3. **极限翻盘**：底部即将封顶时，一个翻转让所有悬浮方块"掉"到侧面，棋盘突然清空大半

## 视觉/音效设计

### 视觉
- **翻转特效**：棋盘 3D 翻转动画（CSS `transform: rotateX/Y` 过渡），持续 0.3 秒
- **重力方向指示**：棋盘四边的发光箭头，当前方向脉动闪烁
- **翻转权计数**：悬浮在侧边的能量槽，每次获得时充能动画
- **方块入场**：根据当前重力方向，方块从对应的边缘滑入
- **背景变化**：四向重力对应四种背景色调（下=深蓝、上=紫色、左=青色、右=橙色）

### 音效
- **翻转音效**：低沉的"嗡——"声，类似电梯启动，随方向有音高变化
- **消行音效**：比标准 Tetris 更响亮的"咔嚓"声，附带重力变化后的回音
- **翻转权获得**：清脆的充能"叮"声，连续获得时有上升音阶
- **BGM**：随重力方向改变调性（下行=大调、上行=小调、左右=五声音阶变奏）

## 技术实现要点

### Canvas 渲染
- 棋盘坐标系变换：维护 `gravityDir: {x, y}` 向量，所有方块移动/锁定逻辑基于此
- 方块入场位置根据重力方向计算：下→`(col, -1)`，上→`(col, 20)`，左→`(-1, row)`，右→`(21, row)`
- Ghost piece 计算方向反转
- `requestAnimationFrame` 主循环，60fps 渲染

### 输入处理
- 方向键：根据当前重力方向映射到棋盘上的"前/后/下"
- R 键：翻转重力
- Space：硬降落（朝当前重力方向）
- 移动端：四向虚拟摇杆 + 翻转按钮

### 状态管理
```javascript
const state = {
  board: Array(20).fill(null).map(() => Array(10).fill(0)),
  gravityDir: { x: 0, y: 1 },  // 初始向下
  flipPoints: 0,                // 累积翻转权
  currentPiece: Piece,
  phase: 1,                     // 1=180度, 2=四向
  level: 1,
  // ...
};
```

### 复用现有代码
- 7-bag 随机器、方块形状定义、消行检测可直接复用 `tetris.html`
- 粒子系统、连击倍率系统可复用
- 需要重写的：方块下落逻辑（方向相关）、Ghost piece 计算、入场动画

## 预估代码量和开发优先级

- **预估代码量**：~1200 行
- **开发优先级**：P0（机制最颠覆，差异化最高，实现路径清晰）
- **预估工期**：2 天
- **文件**：`games/tetris_gravity.html`

## 与现有 tetris.html 的关系

**独立文件**，而非扩展。因为重力翻转需要重写核心下落逻辑，与现有元素方块的特殊方块/法术系统融合复杂度太高。但以下模块可提取为共享工具：
- 方块形状定义（7 种标准 Tetromino）
- 7-bag 随机器
- 粒子系统
- 连击/计分系统
