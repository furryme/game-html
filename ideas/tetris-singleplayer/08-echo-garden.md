# #8 回声花园 ★★★☆☆

> 排名：#8 / 10 | 交叉评分：留存 ★★★ / 玩法 ★★ / 美学 ★★
> 工作量：中 | 推荐：作为 #1 赛季传说的"花园篇"视觉主题使用

---

## 一句话

每块方块是一粒种子，消除一行即花朵绽放，永久花园留存

## 核心要解决的问题

如何将"消除"这一**破坏性**动作彻底翻转为**创造**？

## 核心机制

### 1. 消除=创造

满行消除不再"消失"，而是方块在消行瞬间转化为花朵。7 种方块对应 7 种植物：

| 方块 | 对应植物 | 视觉特征 |
|------|---------|---------|
| I | 向日葵 | 黄色、长条花瓣 |
| O | 蒲公英 | 圆形、白色绒球 |
| T | 三色堇 | 紫色、三角形 |
| S | 蕨类 | 绿色、波浪形 |
| Z | 枫叶 | 红色、对角形 |
| J | 薰衣草 | 蓝紫色、弯钩形 |
| L | 郁金香 | 橙红色、弯钩形 |

消行动画：方块从消行位置向上"生长"为对应的花朵 SVG 矢量图（0.5 秒动画），然后花朵被"采集"并送入永久花园。

### 2. 永久花园

- 每局消除产生的花朵会被保存到"永久花园"中
- 花园是一个独立的可视化区域，以网格布局展示已收集的所有花朵
- 花朵数量随游戏进度增长，形成一个逐渐丰富的私人花园
- 花园不影响方块、消除或策略——纯粹是"消除行数的可视化计数器"
- 使用 `localStorage` 存储花园数据（花朵类型 + 收集时间戳）

### 3. 组合效果（可选升级方向）

- 不同植物相邻时产生额外的视觉效果（如向日葵 + 蒲公英 = 花粉飘散粒子）
- 花园达到一定规模后解锁新的方块皮肤变体
- 可设置"花园目标"（如"收集 100 朵向日葵"）

## "Aha" 时刻

满行消除不再意味着"消失"，而是"绽放"——心理模型完全翻转。类似《Stardew Valley》的种植满足感。

## 致命缺陷

1. 机制完全沿用传统方块规则，留存价值约等于一个好看的成就墙
2. 花园种满后无后续目标设计
3. 需要 7 种植物的矢量美术资源

推荐作为 **#1 赛季传说** 的"花园篇"视觉主题使用，与 **#9 声波共鸣** 捆绑。

## 技术实现要点

- **花朵 SVG**：使用内联 SVG 字符串，7 种植物各一个矢量图（约 30-50 行/植物）
- **生长动画**：消行时从消行行位置创建花朵元素，使用 CSS `transform: scale(0→1) + translateY` 生长动画
- **花园网格**：Canvas 2D 绘制，每朵花占 40x40px 格子，按收集时间顺序从左到右排列
- **花园存储**：`localStorage.setItem('garden', JSON.stringify(flowerList))`，`flowerList` 为 `[{type, collectedAt}]` 数组
- **组合效果**：遍历相邻花朵对，检查 `(typeA, typeB)` 是否在预设组合表中，匹配则播放粒子效果

## 与列表中其他创意的关系

- 作为 **#1 赛季传说** 的"花园篇"视觉主题运行时，机制创新密度提升
- 与 **#9 声波共鸣** 捆绑：不同植物对应不同音高，花园绽放时播放和弦
- 与 **#7 晶体共振** 组合：方块着地时为种子（暗淡），消行后转化为晶体花朵（发光）
