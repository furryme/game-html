# #9 声波共鸣 ★★★☆☆

> 排名：#9 / 10 | 交叉评分：需与视觉方案捆绑使用
> 工作量：小 | 推荐：与 #7 晶体共振或 #8 回声花园捆绑

---

## 一句话

方块映射音符，消行演奏和弦，连击构成旋律

## 核心要解决的问题

如何让一局操作本身就成为一首**乐曲**，而不是背景音效？

## 核心机制

### 1. 方块→音符映射

7 种方块对应 7 个音符（使用**五声音阶**，保证任意组合都和谐）：

| 方块 | 音符 | 频率 (Hz) |
|------|------|----------|
| I | Do | 261.63 |
| O | Re | 293.66 |
| T | Mi | 329.63 |
| S | Sol | 392.00 |
| Z | La | 440.00 |
| J | Do' | 523.25 |
| L | Re' | 587.33 |

方块着地时播放对应音符（短促的 piano 音色，0.1 秒）。消行时播放该行所有方块音符的**和弦**（同时演奏，0.5 秒渐弱）。

### 2. 连击旋律

- 连续消行时，每次消行的和弦按时间轴排列形成一段旋律
- 四消时触发特殊的"解决和弦"（大三和弦上行）
- 连击间隔超过 3 秒则旋律重置
- 连击数影响和弦的八度（连击越高，八度越高）

### 3. 程序化音频

- 使用 Web Audio API 的 `OscillatorNode` 和 `GainNode` 生成，**无需外部音频文件**
- 方块着地音效：正弦波 + 短促衰减（0.1 秒 `GainNode.exponentialRampToValueAtTime`）
- 背景环境音：低频率的持续低音（50Hz 正弦波，`GainNode.gain.value = 0.03`）
- 玩家可在设置中调节音量或完全静音

## "Aha" 时刻

一局优秀的操作本身就是一首乐曲。你不需要看回放，听声音就能感受到那局的高光时刻。

## 致命缺陷

单独作为方案产出区分度不足，需要与视觉方案捆绑使用。

## 关键设计红线

1. 五声音阶保证任意组合和谐（避免不协和音程）
2. 音效必须有 `GainNode` 渐弱控制，避免高频操作时音频堆积
3. 必须有静音开关

## 技术实现要点

- **Web Audio API**：`AudioContext` 单例，方块着地/消行时创建 `OscillatorNode` + `GainNode`
- **和弦合成**：消行时遍历该行方块类型，对每个方块创建对应的 `OscillatorNode`，同时播放
- **连击追踪**：`lastClearTime` 变量，每次消行时检查 `(now - lastClearTime) < 3000` 判定连击
- **四消解决和弦**：四消时额外叠加一个大三和弦（C-E-G 三个 `OscillatorNode`）
- **性能**：`OscillatorNode` 创建/销毁开销低，可同时管理数十个音符

## 与列表中其他创意的关系

- 与 **#7 晶体共振** 组合：折射角度对应音高，堆叠高度决定和声复杂度——视听统一隐喻
- 与 **#8 回声花园** 组合为"花园篇"赛季的完整视听主题：不同植物对应不同音符，花园绽放时播放和弦
- 可与 **#4 记忆碎片** 组合：音频混响参数随叙事章节演变
- 可与 **#1 赛季传说** 的"故障篇"组合：Glitch 效果同时影响音频（bitcrush 式失真）
