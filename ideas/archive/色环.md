# F5 色环 — 颜色匹配

> 原始概念：色彩为核心机制，视觉愉悦。
> 综合评分：★★★★★（全场最高）| 预估代码量：~650 行 | 开发优先级：P1

---

## 一句话描述

旋转色环上的彩色区块缓缓经过——在目标颜色转到指定位置时轻点捕获。手大部分时间不碰屏幕，等待本身就是玩法。

---

## 核心玩法

### 操作（1 个输入）

- **点击**：当目标颜色的区块转到色环上方指定位置时，轻点捕获
- 手在大部分时间**不碰屏幕**——等待→点→等待→点

### 30 秒体验

1. 打开游戏：一个圆形色环在屏幕中央缓慢旋转，上面分布着彩色区块
2. 色环上方有一个目标指示器，显示当前需要捕获的颜色
3. 等待目标颜色区块转到指示器位置→点击→颜色"流"入指示器，色环上该区块缩小
4. 捕获成功后，色环加速一点，同时可能出现新颜色区块
5. 第 4 轮：色环**突然反转旋转方向**（"逆向轮"机制，不提前通知）。玩家需学习预判反转时机
6. 继续等待→点→捕获。颜色周期控制在 2-3 秒，不会无聊

### "逆向轮"机制

每隔 3 轮色环突然反转旋转方向。不提前用文字告知，而是通过色环边缘的渐变方向暗示即将到来的反转。玩家从"被突袭"到"学会预判"，形成"啊，要反转了"的快感。

- 不增加状态管理复杂度（只需一个布尔值和间隔计数器）
- 用极低复杂度创造规则层面的机制深化

### 颜色融合

连续捕获两种在色轮上相邻的颜色（如红→橙），触发"融合"效果——两种颜色混合成新颜色，给予额外奖励。鼓励玩家有策略地选择捕获顺序，而非机械地按提示操作。

### 无失败惩罚

点错颜色不会产生"错误"反馈。颜色区块只是滑过去，色环继续旋转。没有倒计时，没有"扣分"。玩家随时可以停下来发呆看色环转。

## 解压与哲学

### 解压点

**等待即玩法**。玩家大部分时间手不碰屏幕，这种"等待→点"的节奏天然降低了操作焦虑。从机械分类升维为审美判断——不再是"把红色放红色筐"，而是"让颜色在对的时刻归位"。

### 哲学内核

**万物各归其位**（斯多葛哲学"秩序即自由"）。世界之所以让人感到混乱，是因为事物不在它该在的位置。当你把每样东西放到属于它的地方，世界就变得清晰了。

**和谐而非整齐**（Josef Albers 色域理论）。最后一个颜色归位后，整个画面不是整齐而是和谐——差异中的平衡。

### "禅"时刻

经过一段时间的整理，最后一个元素"咔哒"一声归位，整个屏幕从混乱变得色彩分明、层次清晰。那种"完成了"的感觉不是成就感，而是内心的某个结被解开了。

### 情感弧线

混乱（颜色杂乱）→ 辨识（发现规律）→ 和谐（颜色归位）→ 超越（有意不和谐的反转，接受变化）

### 隐喻设计

颜色旋转不会停——你无法控制它何时转到，只能学会在对的时刻行动。暗喻人生中那些"等待时机"的耐心。

## 视觉与声音

### 视觉语言

- **色彩**：7 种离散基本色（HSL 色轮黄金角分布），高饱和但柔和（降 20% 饱和度）。背景接近纯白的浅灰
- **节奏**：中等偏慢，每个匹配动作约 0.5-1 秒。颜色周期 2-3 秒，不催促
- **动效**：匹配成功时元素像水滴落入水面，融入目标区域产生微小涟漪。色环反转时边缘渐变先变向，视觉暗示而非突兀切换
- **参考**：Josef Albers 色域绘画理论，每个色块边缘带水彩晕染

### 声音设计

- 每种颜色对应 C 大调音阶中的一个音符
- 匹配成功时发出该音，连续匹配形成旋律
- 玩家不知不觉"演奏"了一段音乐
- 整体听感：卡洪琴或木琴——温暖、木质、不尖锐
- 作为过程反馈，非成就奖励

## 技术方案

| 项目 | 方案 |
|------|------|
| 核心算法 | HSL 色轮黄金角配色 + 时机判断 |
| 颜色系统 | 7 种离散基本色，HSL 空间控制色差 |
| 色环渲染 | Canvas 2D 圆弧绘制，`requestAnimationFrame` 旋转 |
| 碰撞检测 | 角度差判断（色环角度 vs 指示器位置） |
| 逆向轮 | 布尔值 + 间隔计数器，反转前渐变方向暗示 |
| 颜色融合 | 相邻捕获检测（色轮角度距离 ≤ 60°），混合 HSL |
| 代码量 | 核心 ~350 行 + 基础设施 ~300 行 |
| 移动端适配 | 色环增大，点击区域放大，触摸精度友好 |

### 关键技术要点

- **黄金角配色**：7 种颜色在 HSL 色轮上按黄金角（137.5°）分布，最大化视觉区分度
- **时机窗口**：容错角度 ±15°，移动端可适当增大到 ±20°
- **颜色周期控制**：基础旋转速度确保目标颜色每 2-3 秒经过一次指示器

## 玩家体验

### 摸鱼伪装度

**全场最高**。手大部分时间不碰屏幕，看起来像在等待什么。屏幕上一个旋转的色环，同事可能以为是屏幕保护程序。

### 上手门槛

零教学。看着色环转，发现目标颜色到位置了点一下——直觉就能理解。

### "再来一局"诱因

- 等待→点的节奏本身有冥想感
- "逆向轮"创造了"预判反转"的快感
- 颜色融合鼓励尝试不同捕获顺序
- 无失败压力，随时可停

### 传播潜力

- 色卡导出：将一次游戏的配色方案导出为色卡图片
- 旋律录制：连续匹配形成的旋律可以回放
- 截图即海报：色环旋转的截图具有极简美学

### 长期留存

- "每次打开都有不同"的随机配色，去除 FOMO
- 颜色融合增加策略层
- 逆向轮的反转时机和频率可调

### 注意事项

- 颜色周期不超过 3 秒，避免无聊流失
- 等待期间增加微妙粒子效果填补视觉空白
- 玩法过于单一，长期留存仅靠"逆向轮"不够 → 颜色融合和旋律系统是补充变化

---

## 关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 核心玩法 | 时机判断，非空间归类 | 与已实现游戏无时空结构重合 |
| 摸鱼优势 | 手不碰屏幕 > 碰屏幕 | 等待本身就是摸鱼优势 |
| 机制深化 | 逆向轮，非参数变化 | 低复杂度创造规则深度 |
| 失败处理 | 无惩罚，颜色滑过 | 解压游戏不需要挫败 |
| 颜色系统 | 7 色黄金角，非管道旋转 | 管道视觉像"修水管"，伪装度差 |
| 传播 | 体验质量驱动，非 FOMO | 排行榜和每日挑战与解压矛盾 |
