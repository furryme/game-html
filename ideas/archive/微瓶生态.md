# F3 微瓶生态 — 极简模拟

> 原始概念：迷你文明/生态，一个手指操控，上帝视角的掌控感。
> 综合评分：★★★★☆ | 预估代码量：~950 行 | 开发优先级：P4

---

## 一句话描述

在一个微型瓶子里放置生物，看着它们自行繁衍、竞争、共生——你的每一次干预都像蝴蝶振翅，涟漪扩散。

---

## 核心玩法

### 操作（2 个输入）

- **点击**空白格：放置当前选中的元素类型
- **底部工具栏切换**：滑动/点击切换元素类型（水 / 草 / 动物 / 岩石）
- **长按**：删除已有元素（300ms 确认动画，防止误触）

### 30 秒体验

1. 打开游戏：100x100 网格的世界，Simplex Noise 生成的初始地形（少量水域和草地）
2. 放置第一滴水：水域周围格子开始变成浅蓝，水分缓慢扩散
3. 在水源旁种草：草在湿润格子中生长，绿色色块向外蔓延
4. 放置第一只动物：小动物图标出现，在草地附近移动、吃草、繁殖
5. 观察：草→动物的食物链开始运转，顶部平衡条实时显示 O2 / 养分 / 种群比例的宏观态势
6. 调整：动物太多草不够吃时，放置岩石阻挡扩张，或增加水源促进草地恢复

### 生态规则（元胞自动机）

| 元素 | 行为 |
|------|------|
| 水 | 向相邻空地缓慢扩散（概率 10%/tick） |
| 草 | 需要相邻水才能生长，向空地蔓延（概率 20%/tick） |
| 动物 | 在草上方移动，吃草繁殖，无草时死亡 |
| 火 | 随机点燃干燥草地，向相邻可燃格子蔓延 |
| 岩石 | 不可变，阻挡扩散 |

**死亡即转化**：动物死亡后格子变回空地，但留下养分（加速草的再生）。火的蔓延不导致全盘消亡——烧毁区域变成肥沃土壤，草的再生速度翻倍。这不是"Game Over"，是生态循环。

### 时间加速

1x / 4x / 16x 三档加速。加速时世界边缘有模糊光晕效果。16x 下整个生态的日/夜周期在几秒内完成。

## 解压与哲学

### 解压点

**微观掌控感 + nurturing（养育）渴望**。以"上帝视角"在一个极小的世界里放置元素，系统自动模拟互动生态。"播下一颗种子，看着它自己生长"的体验满足了养育的原始渴望，同时零失败惩罚消除了压力。

### 哲学内核

**蝴蝶效应与无为而治**（道家）。你不需要精确控制每一个细节，只需做出一个微小的干预，系统会自动演化出丰富的生态。最好的管理是最少的干预。

**胜利条件不是世界有多繁荣，而是"玩家多久没有干预"**——从操控者到守护者，最终成为旁观者。

### "禅"时刻

玩家放置了一个看似无关的元素后，退后观察，发现世界开始"活"了——植物向着水源生长，小动物在树荫下出现，整个迷你生态自成循环。那种"我好像创造了一个生命"的感动。

最终，放手后世界自行运转，玩家意识到"它不需要我了"。

### 情感弧线

放置（掌控）→ 观察（好奇）→ 干预（焦虑）→ 退后（信任）→ 放手（平静）

### 隐喻设计

游戏机制暗喻了"养育"这一人生阶段——你无法永远控制孩子的每一步，但你可以创造一个环境，让他们自己成长。

## 视觉与声音

### 视觉语言

- **色彩**：以绿色系为主调（深浅不同），搭配少量蓝色（水）和棕色（土地），天然和谐
- **节奏**：极其缓慢，一格的变化需要 2-3 秒才能看到效果。玩家必须"等"，等待本身就是冥想
- **动效**：元素出现用"生长"动画（从小变大、从透明到实心），消失用"枯萎"动画（缩小、褪色、转化为土壤）
- **容器**：不可见边界（地平线式），无玩具感

### 声音设计

- 底层：持续风声（几乎听不见）
- 种元素：鸟叫一声（草）/ 细流潺潺（水）
- 生态丰富时：声音层叠成"自然白噪音"
- 死亡：不发出负面音效，而是风声短暂加强（暗示"回归自然"）

## 技术方案

| 项目 | 方案 |
|------|------|
| 核心算法 | 元胞自动机（5 类型态）+ **8x8 block-based 分块更新** |
| 网格规模 | 100x100（渲染到 400x400 canvas，4px/格） |
| block 数量 | 13×13 = **169 块**，Uint8Array[169] 存储 isDirty |
| 更新策略 | 只遍历 dirty block 内 64 格；边缘依赖检查延迟传播 |
| 初始世界 | Simplex Noise 3D 生成地形 + Poisson Disc 分布动物 |
| 时间加速 | 1x / 4x / 16x |
| 性能 | 稳态下仅 10-20 个 dirty block，16x 加速仍保持 30fps |
| 代码量 | 核心 ~620 行 + 基础设施 ~330 行 |
| 移动端适配 | 网格 12x12（约 20px/格），触摸放置，工具栏增大 hit area |

### 关键性能优化

**Block-based 分块更新**优于 dirty Set 的原因：
- 冷启动和火蔓延雪崩时 Set 退化到全图，block 上限固定 169
- Uint8Array[169] for 循环比 Set iterator 快 3-5x（V8 引擎优化差异）
- 边缘延迟传播：更新后检查边界依赖，有依赖才标记相邻 block，稳态再减 30-40% dirty

## 玩家体验

### 摸鱼伪装度

**高**。屏幕看起来像动态数据可视化或环境监测面板。

### 上手门槛

30 秒内触发"啊哈时刻"：放水→种草→看草生长。但前 30 秒如果演化太慢，流失率极高。需要初始地形有一定的"种子"来加速第一印象。

### "再来一局"诱因

"再看一眼"的拖延感。模拟游戏独有的粘性——想看看下一轮生态会演变成什么样。

### 传播潜力

延时回放导出短视频，展示生态从空白到繁荣的演化过程。种子系统让有趣局面可复现、可分享。

### 长期留存

- 随机种子地形保证每次开局不同
- 火事件增加动态变化
- 新物种可随版本扩展

### 注意事项

"死亡即结束"改为"死亡即转化"。死亡不是终点，是形态转换。避免玩家因害怕生态死亡而不敢玩。单局控制在 3-5 分钟，死亡成本低。

## 关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 网格大小 | 100x100 + block 分块 | 40x30 丢失涌现行为；dirty Set 雪崩退化 |
| 死亡处理 | 死亡即转化，非 Game Over | 解压游戏不需要挫败感；工程补丁不如哲学框架 |
| 食物网可视化 | 砍掉，改平衡条 | 微观连线 + 宏观色块混用导致认知混乱 |
| 时间加速 | 必须，3 档 | 等待演化是冥想也是无聊的关键平衡点 |
