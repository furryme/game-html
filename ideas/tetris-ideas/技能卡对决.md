# 技能卡对决

**一句话：** 1v1 方块对战 + 每消 4 行抽技能卡（冻结/偷换/炸弹），关键时机出牌逆转局势。

| 维度 | 评分 |
|------|------|
| 好玩度 | ⭐⭐⭐⭐⭐ 方块节奏 + 卡牌博弈，"差一点就赢了然后被反杀"的张力 |
| 传播力 | ⭐⭐⭐⭐ "方块+卡牌"跨界融合，对战录屏精彩，"那局我用炸弹翻盘了" |
| 新意度 | ⭐⭐⭐⭐ 卡牌博弈叠加方块对战，策略深度大幅提升 |
| 实现难度 | ⭐⭐⭐⭐ 中高，双棋盘同步 + 卡牌系统 + AI 对手 |

## 核心机制详解

### 对战基础
- 双棋盘并排显示，左侧玩家，右侧 AI 对手
- 标准 Tetris 对战规则：消行向对手底部发送垃圾行
- 消 2 行发 1 行垃圾，消 3 行发 2 行，消 4 行发 4 行
- 垃圾行中随机有一个空格（位置由 AI 选择）

### 卡牌系统

**抽卡机制：**
- 每消 4 行累积 1 张卡牌，手牌上限 3 张
- 开局赠送 1 张基础卡（冻结）
- 卡牌池共 8 种，每局随机抽取，不可选择

**卡牌列表：**

| 卡牌名 | 效果 | 稀有度 | 冷却 |
|--------|------|--------|------|
| 🧊 冻结 | 对手棋盘冻结 3 秒，无法操作 | 普通 | 无 |
| 💣 炸弹 | 随机消除对手 3 行方块 | 普通 | 无 |
| 🔄 偷换 | 偷取对手的下一块放入自己队列 | 稀有 | 2 行 |
| 🌋 熔岩 | 对手棋盘底部升起 2 行熔岩（不可消除） | 稀有 | 无 |
| 🛡️ 护盾 | 抵消下一次收到的垃圾行 | 普通 | 无 |
| 👻 幽灵 | 自己的当前方块变为穿透模式，可穿过已有方块 | 稀有 | 无 |
| ⏪ 倒带 | 回退自己的棋盘到 5 秒前的状态 | 史诗 | 8 行 |
| 💥 清屏 | 清除自己所有方块但给对方发 5 行垃圾 | 史诗 | 12 行 |

**出牌时机：**
- 任何时候可以点击手牌出牌，但出牌后当前方块立即锁定（掉到底部）
- 手牌以折叠式 UI 显示在棋盘下方，点击展开/出牌
- 出牌后有 0.5 秒的特效播放时间，期间双方都无法操作

### AI 对手
- AI 使用贪心算法放置方块（优先填满行）
- AI 出牌策略基于"威胁评估"：棋盘高度 > 70% 时用护盾，玩家连消时用冻结打断
- AI 难度随玩家等级递增，高等级 AI 会使用"偷换"等高级策略
- AI 手牌在棋盘上方以背面显示，玩家可以看到张数但不知道内容

### 局外成长（可选）
- 每胜利一场获得经验值，升级解锁新卡牌加入卡池
- 卡牌统计面板：使用次数、胜率等
- 数据存储：localStorage，关闭浏览器后保留

## 成瘾循环

消行攒卡 → "这张牌什么时候用"的策略等待 → 关键时机出牌逆转局势 → 对手反制 → 博弈升级 → 翻盘或被翻盘

核心张力：**消行速度的快感 vs 出牌时机的克制**——太早出牌浪费，太晚被对手压制

## "啊哈"时刻

1. **第一次冻结**：对手即将消 4 行，你打出冻结卡，对手方块掉到底部直接 Game Over
2. **偷换翻盘**：对手下一块是长条 I，你偷过来填满了底部，同时对手收到你的垃圾行
3. **护盾反杀**：对手连消发垃圾行，你的护盾抵消，同时你的炸弹消除对手 3 行
4. **史诗清屏**：棋盘即将封顶，清屏卡一键清空并给对手 5 行垃圾，绝地翻盘

## 视觉/音效设计

### 视觉
- **双棋盘布局**：桌面端左右并排，移动端上下堆叠
- **卡牌 UI**：底部横向排列的卡牌，点击时弹起放大并显示效果说明
- **出牌特效**：
  - 冻结：对手棋盘覆盖冰霜纹理，3 秒后碎裂
  - 炸弹：对手棋盘中心爆炸，冲击波向外扩散
  - 偷换：两块棋盘之间的方块交换动画
  - 熔岩：对手底部升起红色熔岩，带气泡粒子
- **垃圾行发送**：从己方棋盘飞出红色方块流，冲击对手棋盘底部
- **卡牌稀有度**：普通=白色边框、稀有=蓝色边框、史诗=金色边框

### 音效
- **抽卡音效**：卡牌翻开的"唰"声，稀有卡牌有额外闪光音效
- **出牌音效**：根据卡牌类型不同
  - 冻结：冰块碎裂声
  - 炸弹：爆炸声
  - 偷换：嗖——的传送声
- **垃圾行攻击**：低频冲击声
- **护盾激活**：盾牌格挡的"铛"声
- **BGM**：紧张的对战电子乐，出牌时加入重音

## 技术实现要点

### 双棋盘同步
- 两个独立的游戏状态，共享同一个渲染循环
- 玩家棋盘：键盘控制 + 卡牌 UI 点击
- AI 棋盘：自动放置 + 自动出牌逻辑
- 垃圾行发送：异步通信，一方消行 → 另一方收到

### 卡牌效果系统
```javascript
const cardEffects = {
  freeze: (targetBoard, duration) => { targetBoard.frozen = duration; },
  bomb: (targetBoard) => { /* 随机消除3行 */ },
  steal: (sourceBoard, targetBoard) => {
    const piece = sourceBoard.nextQueue.shift();
    targetBoard.nextQueue.push(piece);
  },
  // ...
};

function playCard(card, playerBoard, opponentBoard) {
  cardEffects[card.type](playerBoard, opponentBoard);
  playerBoard.currentPiece.lock(); // 出牌后立即锁定
  playCardAnimation(card.type);    // 0.5秒特效
}
```

### AI 决策树
1. **放置决策**：贪心算法计算每种放置的消行数和棋盘高度，选择最优
2. **出牌决策**：
   - 如果棋盘高度 > 70% 且有护盾 → 出护盾
   - 如果玩家连续消行且有冻结 → 出冻结
   - 如果手牌满 3 张且无防御牌 → 出进攻牌
   - 其他情况 → 继续攒卡

### 移动端适配
- 双棋盘上下堆叠，每张 8×16 缩小网格
- 卡牌 UI 为底部滑出面板，点击展开
- 触摸控制：虚拟方向键 + 旋转按钮
- 屏幕空间紧张时，AI 手牌只显示张数

### 复用现有代码
- 方块核心逻辑（下落、旋转、消行）可复用
- 7-bag 随机器、Ghost piece 可复用
- 需要新增：卡牌系统、AI 对手、双棋盘管理、出牌特效

## 预估代码量和开发优先级

- **预估代码量**：~1400 行
- **开发优先级**：P2（策略深度高，但 AI + 卡牌系统增加复杂度）
- **预估工期**：3 天
- **文件**：`games/tetris_battle_cards.html`

## 与现有 tetris.html 的关系

**独立文件**。双棋盘对战 + 卡牌系统是全新架构，与元素方块的单棋盘/法术系统设计方向不同。但方块渲染、消行逻辑、输入处理等基础模块可提取复用。
