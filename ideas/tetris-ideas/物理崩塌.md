# 物理崩塌

**一句话：** 方块触底后不是立刻锁死，而是进行"支撑检查"——下方无支撑的方块逐格下落，可能连锁倒塌。

| 维度 | 评分 |
|------|------|
| 好玩度 | ⭐⭐⭐⭐⭐ 从"精确堆叠"变成"危险建筑"，连锁倒塌带来极强视觉爽感 |
| 传播力 | ⭐⭐⭐⭐ "多米诺骨牌"式连锁倒塌，截图和录屏传播价值高 |
| 新意度 | ⭐⭐⭐⭐ 在传统 Tetris 上叠加结构稳定性维度，独特但不颠覆 |
| 实现难度 | ⭐⭐⭐ 中，确定性"延迟重力"无需物理引擎 |

## 核心机制详解

### 支撑系统
- 方块触底或落在其他方块上后，不是立刻锁定，而是进入"暂稳"状态
- **支撑规则**：一个方块必须满足以下任一条件才能保持不动：
  1. 下方有地板
  2. 下方有其他方块（已锁定或暂稳）
  3. 左右两侧都有方块且下方无空缺（"架桥"结构）
- 不满足支撑条件的方块，每帧向下坠落一格
- 坠落过程中可能撞到其他方块，触发连锁崩塌

### 延迟重力
- 方块"暂稳"后有 **1.5 秒的宽限期**，宽限期内玩家可以调整其左右位置
- 宽限期结束后，方块进行第一次支撑检查
- 此后**每次消行或新方块锁定后**，都会触发全棋盘支撑检查
- 支撑检查是确定性的（非物理引擎）：逐格扫描，无支撑则下落，直到所有方块稳定

### 崩塌连锁
- 一个方块下落后，原本在它上方的方块可能失去支撑，继续下落
- **连锁上限**：每轮崩塌最多下落 5 格（防止全屏崩塌的挫败感）
- 崩塌过程中下落的方块撞到锁定的方块时产生碰撞特效和音效
- 如果崩塌导致某些行被填满，也会触发消行（消行再触发支撑检查，形成循环）

### 特殊方块
- **基石方块**（10% 概率）：黄色方块，锁定后永远不需要支撑，可作为建筑结构的基础
- **脆弱方块**（15% 概率）：半透明方块，被其他方块撞击时会被消除而非推开
- **胶合方块**（5% 概率）：粉色方块，锁定后会为相邻方块提供"侧面支撑"，增强结构稳定性

## 成瘾循环

放置方块 → 宽限期内微调位置 → 支撑检查 → 部分结构崩塌的紧张感 → 连锁倒塌的视觉爽感 → 崩塌可能填满新行触发消行 → 消行引发新一轮支撑检查 → 循环

核心张力：**搭建复杂结构（消更多行）vs 结构稳定性（崩塌风险）**

## "啊哈"时刻

1. **第一次崩塌**：你精心搭建的塔楼因为底部消行而整体倒塌，方块如多米诺骨牌般坠落
2. **崩塌消行**：倒塌的方块在下落过程中意外填满了两行，触发消行，连锁继续
3. **基石妙用**：在底部放一个基石方块，然后在其上搭建一个完整的"拱门"结构，两侧同时消行时结构不塌
4. **胶合救援**：濒临倒塌时，一个胶合方块救住了整个结构，从"危险建筑"变成"稳固城堡"

## 视觉/音效设计

### 视觉
- **暂稳状态**：方块在宽限期内轻微晃动（±1px 的抖动动画），颜色半透明
- **支撑检查**：全棋盘闪烁一帧红色扫描线，然后无支撑方块开始下落
- **崩塌动画**：下落的方块加速下落（0.05 秒/格 → 0.02 秒/格），尾部带拖影粒子
- **碰撞特效**：方块撞到锁定方块时产生小型爆炸粒子，白色碎屑
- **基石方块**：底部有发光效果，类似石质纹理（CSS 渐变模拟）
- **脆弱方块**：半透明且边缘有裂纹效果
- **胶合方块**：粘性光晕，相邻方块被"拉住"的视觉效果

### 音效
- **支撑检查**：低沉的"嗡"声，类似地震前兆
- **方块下落**：每次下落一格都有"咔嗒"声，连续下落时音速递增
- **碰撞音效**：清脆的撞击声，力度根据下落速度变化
- **崩塌消行**：比标准消行更响亮的爆炸声
- **基石放置**：厚重的"咚"声，类似大鼓

## 技术实现要点

### 支撑检查算法
```javascript
function checkSupport(grid) {
  let unstable = [];
  for (let y = 0; y < ROWS; y++) {
    for (let x = 0; x < COLS; x++) {
      if (!grid[y][x]) continue;
      if (!hasSupport(grid, x, y)) {
        unstable.push({ x, y });
      }
    }
  }
  return unstable;
}

function hasSupport(grid, x, y) {
  // 下方有地板
  if (y === ROWS - 1) return true;
  // 下方有方块
  if (grid[y + 1][x]) return true;
  // 架桥：左右都有方块且相邻方块自身有支撑
  if (x > 0 && x < COLS - 1 && grid[y][x - 1] && grid[y][x + 1]) {
    return hasSupport(grid, x - 1, y) && hasSupport(grid, x + 1, y);
  }
  return false;
}
```

### 崩塌动画
- 使用帧间延迟递增实现加速下落效果
- 每格 0.05 秒开始，每格减少 0.005 秒，最低 0.02 秒
- 崩塌期间暂停玩家输入，防止操作干扰
- 崩塌结束后恢复游戏

### 消行 + 支撑检查循环
1. 新方块锁定 → 触发支撑检查
2. 崩塌完成 → 检测消行
3. 消行 → 再次触发支撑检查
4. 重复直到棋盘稳定（无崩塌且无消行）

### 性能优化
- 棋盘 10×20 = 200 格，支撑检查 O(N) 复杂度完全可以接受
- 架桥递归使用记忆化避免重复计算
- 崩塌连锁上限 5 格限制递归深度

### 复用现有代码
- 方块形状、7-bag 随机器、消行检测、粒子系统可复用
- 需要新增：支撑检查系统、崩塌动画、特殊方块逻辑

## 与现有 tetris.html 的关系

**独立文件**。支撑系统需要改造方块锁定逻辑，与现有特殊方块系统难以兼容。但核心渲染/输入/计分模块可复用。
