# 高清皮肤 — 精灵图渲染系统

## 概述

在现有 100% 程序化像素渲染的基础上，增加**精灵图（PNG sprite sheet）**作为可选的皮肤系统。现有像素风完全保留，高清皮肤作为主题系统的扩展模块叠加。

**目标**：玩家/敌人/Boss 用 PNG 精灵图替换像素字符串，地图瓦片和 UI 暂不改动。

**核心思路**：高清图 → 转换脚本缩小为像素精灵图 → 游戏加载 PNG 替代字符串绘制。

---

## 当前渲染架构

### 精灵系统

| 元素 | 网格尺寸 | PIXEL 缩放 | 渲染尺寸 | 瓦片占用 |
|------|---------|-----------|---------|---------|
| 玩家（地图） | 8×12 | ×4 | 32×48 | 缩放到 16×16 瓦片 |
| 玩家（战斗） | 10×14 | ×4 | 40×56 | 固定 64×64 |
| 敌人（地图） | 8×8 | ×4 | 32×32 | 缩放到 16×16 瓦片 |
| 敌人（战斗） | 8×8 | ×4 | 32×32 | 放大到 64×64 |
| Boss（地图） | 16×16 | ×4 | 64×64 | 缩放到 32×32 |
| Boss（战斗） | 16×16 | ×4 | 64×64 | 放大到 120×120 |

- PIXEL = 4（每个字符串字符 = 4px）
- TILE_SIZE = 16（4×4 PIXEL）
- Canvas 内部分辨率 480×270
- CSS `image-rendering: pixelated` 放大显示

### 精灵定义方式

`js/view/sprites.js` 中精灵用**字符串网格**定义：

```
var WARRIOR_IDLE = [
  '  aaaa  ',
  ' aaaaaa ',
  '  aaaa  ',
  '  XaXaX ',
  ...
];
```

每个字符映射调色板索引（0-9, a-z → 0-35），`parseSprite()` 转数字数组，`drawSprite()` 逐像素 `fillRect(1px)` 绘制。

### 主题系统

`js/core/theme.js` 支持 6 主题 × 6 模块（palette/sprites/tiles/ui/effects/audio），深继承 + 模块混装。精灵颜色通过 `getThemeSpritePalette()` 切换。

**当前不可换肤的部分**：精灵几何形状（字符串图案本身）、渲染方法。

## 精灵图格式规范

### 文件命名

```
sprite-assets/
  default/               ← 默认像素风（已有，可选 PNG 版本）
    player-warrior.png   ← 职业_角色.png
    player-mage.png
    player-rogue.png
    enemy-slime.png
    enemy-skeleton.png
    boss-dragon.png
  high-res/              ← 高清转换后的精灵图
    player-warrior.png
    ...
```

### 精灵图布局

**单张 PNG = 一个角色的所有动画帧横向排列**

```
┌─────────┬─────────┬─────────┬─────────┐
│  idle_1  │ idle_2  │ walk_1  │ walk_2  │
│ 32×48    │ 32×48   │ 32×48   │ 32×48   │
└─────────┴─────────┴─────────┴─────────┘
```

- 每帧尺寸 = 当前精灵的渲染尺寸（地图 32×48，战斗 64×64）
- 帧之间无间隙
- 透明背景（PNG alpha）

### 精灵元数据

`js/data/sprite-assets/manifest.json` 描述每个精灵的帧信息：

```json
{
  "player-warrior": {
    "src": "player-warrior.png",
    "frameW": 32,
    "frameH": 48,
    "animations": {
      "idle":   { "frames": [0, 1],       "speed": 30 },
      "walk":   { "frames": [2, 3, 4, 5], "speed": 15 },
      "attack": { "frames": [6, 7, 8],    "speed": 8 },
      "hurt":   { "frames": [9],          "speed": 12 }
    },
    "combatFrameW": 64,
    "combatFrameH": 64,
    "combatAnimations": {
      "idle":   { "frames": [0, 1],       "speed": 30 },
      "attack": { "frames": [2, 3, 4],    "speed": 8 },
      "hurt":   { "frames": [5],          "speed": 12 }
    }
  },
  "enemy-slime": { ... },
  "boss-dragon": { ... }
}
```

### 精灵图与字符串精灵的映射

`manifest.json` 中可以指定每个精灵图对应哪个字符串精灵变量：

```json
{
  "player-warrior": {
    "replaces": ["WARRIOR_IDLE", "WARRIOR_WALK", "WARRIOR_ATTACK", "WARRIOR_HURT"],
    ...
  }
}
```

这样 `drawPlayerSprite()` 查找流程：
1. 主题有 `spriteAssets` 且有对应 PNG → 用 `drawImage()`
2. 否则 → 用现有字符串 + `fillRect()`（零改动回退）

## 高清图转换脚本

### 输入

- 任意尺寸高清图（推荐 256×256 ~ 1024×1024）
- PNG 格式，透明背景
- 可以是 AI 生成的角色立绘、手绘插画、3D 渲染截图

### 处理流程

```
高清图 (512×512)
  ↓ 裁剪/居中到目标比例
  ↓ 缩小到目标尺寸 (32×48)
  ↓ 差值抖动（Floyd-Steinberg）
  ↓ 颜色量化（可选，限制调色板）
  ↓ 输出 PNG
```

### 技术选型

使用 `sharp`（Node.js 图像处理库，快速且质量好）：

```javascript
// tools/sprite-convert.mjs
import sharp from 'sharp';
import { writeFile, mkdir } from 'fs/promises';
import { dirname } from 'path';

async function convertSprite(srcFile, targetSize, outputDir) {
  const { data, info } = await sharp(srcFile)
    .resize(targetSize.w, targetSize.h, {
      fit: 'contain',    // 保持比例，透明背景填充
      background: { r: 0, g: 0, b: 0, alpha: 0 },
    })
    .jpeg({ mozjpeg: true })  // or .png()
    .raw()
    .toBuffer();

  // Floyd-Steinberg 抖动 + 颜色量化（可选）
  // 使用 zxp 或自定义量化算法
  const quantized = quantizeAndDither(data, info.width, info.height, palette);

  await mkdir(outputDir, { recursive: true });
  await writeFile(outputDir + '/output.png', quantized.png);
}

// 批量转换
async function batchConvert(manifest, inputDir, outputDir) {
  for (const [name, meta] of Object.entries(manifest)) {
    await convertSprite(
      inputDir + '/' + name + '.png',
      { w: meta.frameW, h: meta.frameH },
      outputDir
    );
  }
}
```

### 使用方式

```bash
cd tools
npm install sharp   # 一次性安装
node sprite-convert.mjs \
  --input ./high-res-sprites/ \
  --output ../js/data/sprite-assets/high-res/ \
  --manifest ../js/data/sprite-assets/manifest.json
```

### 输出

- 每张高清图 → 对应尺寸的像素精灵图
- 如果高清图包含多帧（横向排列的动画帧），脚本自动分割
- 输出目录直接放到 `sprite-assets/` 中即可被游戏加载

## 实现阶段

### Phase 1: 精灵图加载器（基础架构）

**改动**：新建 `sprite-loader.js`，修改 `sprites.js`

- `SpriteLoader` 类：
  - `load(manifestPath)` — 加载 manifest.json，创建 Image 对象
  - `getSprite(name, animation, frame, isCombat)` — 返回 {image, sx, sy, sw, sh}
  - `isReady(name)` — 是否已加载完成
  - `waitForAll()` — Promise，所有精灵加载完成后 resolve

- `drawSprite()` 增加 Image 分支：
  ```javascript
  function drawSprite(ctx, x, y, pattern, palette, opts) {
    if (pattern instanceof Image || pattern instanceof HTMLCanvasElement) {
      // 精灵图路径
      ctx.drawImage(pattern, x, y, opts.w, opts.h);
    } else {
      // 现有字符串路径（不变）
      var grid = typeof pattern === 'string' ? parseSprite(pattern) : pattern;
      // ... fillRect 逐像素绘制
    }
  }
  ```

- `drawPlayerSprite()` / `drawEnemySprite()` / `drawBossSprite()` 支持 Image 输入：
  - 如果主题有 `spriteAssets` 且对应精灵已加载 → 传入 Image
  - 否则 → 传入字符串（现有行为）

**验证**：
- 加载一组测试精灵图，切换主题后渲染正常
- 精灵图加载失败时自动回退到字符串精灵

### Phase 2: 转换工具

**改动**：新建 `tools/sprite-convert.mjs`

- 实现高清图 → 像素精灵图的转换
- 支持批量转换
- 支持多帧精灵图自动分割

**验证**：
- 输入 512×512 角色立绘 → 输出 32×48 像素精灵图
- 输出 PNG 在浏览器中清晰可见
- 与 manifest.json 帧信息匹配

### Phase 3: 主题集成

**改动**：修改 `theme.js`、`main.js`

- 主题结构增加 `spriteAssets` 模块：
  ```javascript
  var HIGH_RES_THEME = {
    // ... 现有模块
    spriteAssets: {
      manifest: 'js/data/sprite-assets/high-res/manifest.json',
      basePath: 'js/data/sprite-assets/high-res/',
    }
  };
  ```

- `themeSwitch()` 切换时：
  - 如果新主题有 `spriteAssets` → `SpriteLoader.load()`
  - 渲染循环自动使用新精灵图
  - 回退：加载失败 → 自动用字符串精灵

- `initGame()` 预加载默认精灵图

**验证**：
- 切换主题后精灵图实时更新
- 字符串精灵和精灵图之间无缝切换
- 内存不会泄漏（旧精灵图被 GC 回收）

### Phase 4: 动画系统增强

**改动**：修改 `sprites.js` 动画逻辑

- 现有动画：切换字符串数组（`WARRIOR_IDLE[0]` vs `WARRIOR_WALK[0]`）
- 精灵图动画：`getSprite()` 返回当前帧的 `drawImage` 参数
- 帧率控制：`animation.speed` = 每帧停留的 tick 数

**验证**：
- 站立/行走/攻击/受伤动画流畅切换
- 战斗中和地图上动画独立（不同帧尺寸和帧数）

## 关键设计决策

### 精灵图 vs 字符串精灵共存

**原则**：精灵图是**纯增量**，不修改任何现有字符串精灵代码。

```
drawPlayerSprite():
  if 主题.hasSpriteAssets && 精灵图已加载:
    ctx.drawImage(spriteSheet, sx, sy, sw, sh, dx, dy, dw, dh)
  else:
    现有 fillRect 路径（零改动）
```

### 精灵图尺寸

**推荐**：保持当前渲染尺寸不变（地图 32×48，战斗 64×64），只替换内容来源。

高清图转换后**缩小**到目标尺寸，不是直接渲染高清图。这样：
- 不需要改 Canvas 分辨率
- 不需要改坐标计算
- 性能和像素风一致
- 精灵图可以比手绘更精细（来源是高清图，缩小后自然有细节）

### 单文件 vs 多文件

精灵图作为**额外文件**存放，不嵌入 HTML：
- `js/data/sprite-assets/` 目录下存放 PNG
- 通过 `new Image().src` 异步加载
- 不影响无精灵图时的单文件运行
- 发布时用 `zip` 打包整个目录

### 颜色量化策略

**两种模式**：

1. **自由颜色**（默认）：转换后保留原始颜色，不限调色板。精灵图用 PNG truecolor。
2. **调色板限制**（可选）：量化到 34 色（与 `SPRITE_PAL` 一致），保持像素风统一感。

默认用自由颜色，用户可以在转换脚本中指定 `--palette` 参数限制颜色。

## 风险评估

| 风险 | 等级 | 缓解 |
|------|------|------|
| 精灵图加载失败 | 低 | 自动回退到字符串精灵 |
| 精灵图加载慢（大文件） | 低 | 文件极小（32×48 PNG < 2KB），预加载遮罩 |
| 动画帧不同步 | 低 | manifest.json 集中管理帧信息 |
| 内存泄漏（切换主题） | 低 | 旧 Image 对象引用清除，让 GC 回收 |
| 高清图缩小后模糊 | 中 | 转换脚本用差值抖动 + 最近邻缩小 |
| 跨域加载（本地文件协议） | 中 | 精灵图在同源目录，无跨域问题 |
