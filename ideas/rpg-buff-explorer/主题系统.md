# RPG 主题皮肤系统设计文档

> **项目**：地牢勇者 RPG（`dev-1-rpg.html`）
> **视角**：视觉总监
> **目标**：类似手游换皮肤的模块化替换系统，支持运行时切换、混搭、解锁、自定义

---

## 1. 主题系统架构

### 1.1 主题数据格式（JS 对象）

一个主题是一个纯 JS 对象，可序列化/反序列化，无需任何外部依赖。所有数据内联在单文件中。

```javascript
{
  // 元数据
  id: "classic",                    // 唯一标识
  name: "经典地下城",               // 显示名称
  rarity: "common",                 // common / rare / epic / legend
  unlockCondition: null,            // null=默认解锁 | { type: "reach_floor", value: 3 } 等

  // === 模块 1: 调色板 ===
  palette: {
    // 背景
    bgDeep:      "#0a0a14",         // 最深背景（未探索区域）
    bgGame:      "#10101c",         // 面板背景
    bgSurface:   "#1a1a2e",         // 卡片/弹窗背景

    // 地图瓦片色
    wallDark:    "#2a2a3e",         // 墙壁-可见
    wallLight:   "#333349",         // 墙壁高光
    floorDark:   "#1e1e30",         // 地板-可见
    floorLight:  "#222238",         // 地板-棋盘格
    corridor:    "#1c1c2e",         // 走廊

    // UI 色
    textPrimary: "#e0e0e0",
    textSecondary: "#8888aa",
    textMuted:   "#555566",
    gold:        "#ffd700",         // 金色（分数/货币/强调）
    danger:      "#e53935",         // 伤害/危险
    heal:        "#66bb6a",         // 治疗/增益
    magic:       "#7c4dff",         // 技能/魔法
    info:        "#64b5f6",         // 信息提示

    // 功能色
    overlayBg:   "rgba(10,10,20,0.88)",
    borderSubtle:"rgba(255,255,255,0.08)",
    borderActive:"rgba(255,255,255,0.2)",

    // 职业色覆盖（可选，不指定则用默认）
    classColors: {
      warrior: "#e53935",
      mage:    "#7c4dff",
      rogue:   "#66bb6a"
    }
  },

  // === 模块 2: 精灵变体 ===

    // 玩家角色光晕颜色（按职业）

    // 粒子尾迹: true = 拖尾 | false = 无

    floorStyle: "checker",

    corridorStyle: "solid",

    fogColor: "#050510",

    // 已探索但不在视野内的暗度 (0-1)
    revealedDarkness: 0.4,
  },

  // === 模块 4: UI 风格 ===
  ui: {
    // 圆角

    borderRadiusMd: 8,

      rare: "#64b5f6",
      epic: "#ce93d8",
      legend: "#ffd700"

  // === 模块 5: 特效参数 ===
  effects: {
    // 玩家光晕
    glowBlur: 10,
    glowIntensity: 0.6,

    shakeAmplitude: 8,

    particleCountMult: 1.0,

    // 浮動伤害上升速度 (ms)

    // 过场动画时长 (ms)
    transitionDuration: 300,

    // 楼层切换遮罩背景
    floorBannerBg: "rgba(0,0,0,0.85)",
  },

  // === 模块 6: 音效主题（可选扩展）===
  audio: {
    // 音效色调: "bright" | "dark" | "hollow"
    tone: "dark",
    // 背景音乐 BPM
    bpm: 120,
    // 打击音效类型: "metal" | "wood" | "stone"
    hitSound: "metal"
  }
}
```

### 1.2 模块分层与依赖关系

```
主题对象 (Theme)
├── palette       ← 最底层，所有模块都依赖色板
├── sprites       ← 依赖 palette，控制精灵视觉变体
├── tiles         ← 依赖 palette，控制地图瓦片绘制
├── ui            ← 依赖 palette，控制 HTML/CSS UI 组件
├── effects       ← 独立参数，控制动画/粒子参数
└── audio         ← 独立参数，控制音效特征（后续扩展）
```

**模块加载顺序**：`palette → tiles → sprites → ui → effects → audio`

### 1.3 主题继承机制

所有主题继承自一个 `BASE_THEME`，只需覆盖差异部分：

```javascript
const BASE_THEME = { ... };  // 默认经典地下城的完整定义

// 派生主题只需覆盖差异
const BLOOD_THEME = {
  ...BASE_THEME,
  id: "blood_moon",
  name: "血月之夜",
  palette: {
    ...BASE_THEME.palette,
    bgDeep: "#1a0a0a",
    danger: "#ff1744",
    gold: "#ff6e42",
    // ... 其他覆盖
  },
  tiles: {
    ...BASE_THEME.tiles,
    wallStyle: "cave",
    floorStyle: "cracked",
  },
  effects: {
    ...BASE_THEME.effects,
    particleCountMult: 1.3,
    glowBlur: 14,
  }
};
```

继承规则：
- **一级合并**：顶层字段直接覆盖
- **palette 合并**：颜色字段逐个覆盖，未覆盖的保留基类值
- **sprites/tiles/ui/effects 合并**：同 palette，浅合并

### 1.4 注册与切换 API

```javascript
const ThemeManager = {
  _themes: {},           // id -> Theme
  _active: null,         // 当前激活的主题对象
  _layerOverride: null,  // 地城层临时覆盖
  _equippedModules: {},  // 混装模块: { tiles: "void", ui: "classic", ... }

  // 注册一个主题
  register(theme) {
    // 自动继承 BASE_THEME
    const merged = this._deepMerge(BASE_THEME, theme);
    this._themes[merged.id] = merged;
    return merged;
  },

  // 切换主题（整体切换）
  switch(id) {
    const t = this._themes[id];
    if (!t) return console.error('[theme] not found:', id);
    this._active = t;
    this._equippedModules = {};  // 清除混装
    this._apply();
    localStorage.setItem('rpg-theme', id);
  },

  // 混装模式：指定模块来自不同主题
  equip(moduleName, themeId) {
    const source = this._themes[themeId];
    if (!source || !source[moduleName]) return;
    this._equippedModules[moduleName] = themeId;
    this._apply();
  },

  // 地城层临时覆盖（进入某层时调用，离开时清除）
  applyLayerOverride(floorNum, override) {
    this._layerOverride = { floorNum, ...override };
    this._apply();
  },
  clearLayerOverride() {
    this._layerOverride = null;
    this._apply();
  },

  // 获取当前生效的完整主题（合并混装 + 层覆盖）
  getActive() {
    if (!this._active) return BASE_THEME;
    let t = { ...this._active };
    // 1. 混装覆盖
    for (const [mod, srcId] of Object.entries(this._equippedModules)) {
      const src = this._themes[srcId];
      if (src && src[mod]) t[mod] = { ...t[mod], ...src[mod] };
    }
    // 2. 地城层覆盖
    if (this._layerOverride) {
      t = this._deepMerge(t, this._layerOverride.patch);
    }
    return t;
  },

  // 应用到 DOM + Canvas
  _apply() {
    const t = this.getActive();
    this._applyCSS(t);     // CSS Custom Properties
    this._applyCanvas(t);  // Canvas 渲染参数
    dispatchEvent(new CustomEvent('themechange', { detail: t }));
  },

  // CSS 变量注入
  _applyCSS(t) {
    const root = document.documentElement;
    // palette → CSS vars
    for (const [k, v] of Object.entries(t.palette)) {
      root.style.setProperty(`--tp-${k}`, v);
    }
    // ui → CSS vars
    for (const [k, v] of Object.entries(t.ui)) {
      root.style.setProperty(`--tu-${k}`, v);
    }
    // effects → CSS vars
    for (const [k, v] of Object.entries(t.effects)) {
      root.style.setProperty(`--te-${k}`, v);
    }
  },

  // Canvas 参数缓存（供渲染函数读取）
  _applyCanvas(t) {
    CanvasTheme = { ...t.palette, ...t.tiles, ...t.sprites, ...t.effects };
  },

  // 初始化
  init(themeList) {
    // 注册全部主题
    for (const t of themeList) this.register(t);
    // 恢复用户选择
    const saved = localStorage.getItem('rpg-theme');
    this.switch(saved || 'classic');
  },

  // 深合并工具
  _deepMerge(base, override) {
    const result = { ...base };
    for (const [k, v] of Object.entries(override)) {
      if (v && typeof v === 'object' && !Array.isArray(v) && base[k] && typeof base[k] === 'object') {
        result[k] = this._deepMerge(base[k], v);
      } else {
        result[k] = v;
      }
    }
    return result;
  }
};
```

### 1.5 主题对象大小估算

| 内容 | 压缩后大小 | 说明 |
|------|-----------|------|
| BASE_THEME（完整定义） | ~1.2 KB gzip | 所有 6 个模块的完整字段 |
| 派生主题（差异覆盖） | ~300-500 KB gzip  each | 平均每个派生覆盖 15-25 个字段 |
| 6 个主题总计 | ~3-4 KB gzip | 内联到单文件完全可接受 |
| ThemeManager 引擎代码 | ~1.5 KB gzip | 约 100 行 JS |
| **合计** | **~5-6 KB gzip** | 对单文件游戏影响可忽略 |

## 3. 主题 × 地城叠加规则

### 3.1 三层地城基本信息

| 层数 | 名称 | 环境氛围 | 色调方向 |
|------|------|---------|---------|
| 1-2 | 青苔回廊 | 潮湿、自然、古老 | 青绿/石灰 |
| 3-4 | 骨骸 Crypt | 死亡、腐朽、阴冷 | 白骨/暗灰 |
| 5 | 虚空核心 | 混沌、扭曲、危险 | 紫黑/血红 |

### 3.2 叠加模型：全局主题 80% + 地城层 20%

```
最终视觉 = 全局主题(主色/风格/特效) + 地城层微调(色调偏移/环境粒子)
```

**优先级规则**：
1. 全局主题决定 80% 视觉：色板主体、瓦片样式、UI 风格、粒子风格
2. 地城层贡献 20% 微调：墙壁色调偏移、环境粒子颜色、少量额外瓦片纹理

地城层覆盖是**增量补丁**，不改变主题的基本风格。换主题后，地城层仍然有辨识度。

### 3.3 地城层覆盖补丁定义

```javascript
const DUNGEON_LAYER_OVERRIDES = {
  // 第 1-2 层: 青苔回廊
  1: {
    patch: {
      palette: {
```

### 4.2 预览缩略图实现

每个主题卡片的缩略图用一个小 Canvas（80x60 像素）实时渲染：

```javascript
function renderThemeThumbnail(canvas, theme, layer = 1) {
  const ctx = canvas.getContext('2d');
  const t = theme;
  const p = t.palette;

  // 背景
  ctx.fillStyle = p.bgDeep;
  ctx.fillRect(0, 0, 80, 60);

  // 模拟 10x8 的迷你地图
  const ts = 6; // 缩略图 tile size
  for (let y = 0; y < 8; y++) {
    for (let x = 0; x < 10; x++) {
      const isWall = (x === 0 || y === 0 || x === 9 || y === 7 ||
                      (x >= 3 && x <= 5 && y >= 3 && y <= 5));
      if (isWall) {
        ctx.fillStyle = p.wallDark;
        ctx.fillRect(x * ts, y * ts, ts, ts);
      } else {
        ctx.fillStyle = p.floorDark;
        ctx.fillRect(x * ts, y * ts, ts, ts);
        if ((x + y) % 2 === 0 && t.tiles.floorStyle === 'checker') {
          ctx.fillStyle = p.floorLight;
          ctx.fillRect(x * ts + 1, y * ts + 1, ts - 2, ts - 2);
        }
      }
    }
  }

  // 玩家光晕
  ctx.shadowColor = p.classColors?.warrior || '#e53935';
  ctx.shadowBlur = Math.min(t.effects.glowBlur, 6);
  ctx.fillStyle = '#fff';
  ctx.fillRect(4 * ts + 1, 2 * ts + 1, ts - 2, ts - 2);
  ctx.shadowBlur = 0;

  // 楼梯
  ctx.fillStyle = t.sprites.stairGlow;
  ctx.fillRect(8 * ts + 1, 6 * ts + 1, ts - 2, ts - 2);

  // 粒子装饰（静态）
  for (let i = 0; i < 3; i++) {
    ctx.fillStyle = p.danger + '40';
    ctx.fillRect(6 * ts + i * 2, 1 * ts + i * 3, 2, 2);
  }
}
```

### 4.5 获取经济

### 5.1 混装规则（跨主题混搭）

用户可以从不同主题中选取模块自由组合：

```
允许混装的模块: tiles, sprites, ui, effects
不允许混装的模块: palette (色板是基础，混装会冲突)
```

**混装规则**：
1. 先选定一个**基础主题**（提供 palette + 其他所有模块）
2. 在商城底部"混装模式"区域，为每个可混装模块选择来源主题
3. 只有已解锁的主题才能作为混装来源
4. 混装结果实时生效（不保存为独立主题）

**实现示例**：
```javascript
// 基础主题: 经典地下城
ThemeManager.switch('classic');

// 混装: 瓦片来自像素复古, 特效来自虚空核心
ThemeManager.equip('tiles', 'pixel_retro');    // 纯方块瓦片
ThemeManager.equip('effects', 'void_core');    // 双倍粒子 + 大光晕
// → 结果: 经典配色 + 像素瓦片 + 虚空粒子特效
```

**混装组合上限**：同时最多 4 个模块混装（tiles/sprites/ui/effects 各 1 个）。

```javascript
function drawTile(ctx, x, y, tileType, inView) {
  const ct = CanvasTheme;
  const px = x * TILE_SIZE, py = y * TILE_SIZE;

  if (tileType === TILE.WALL) {
    ctx.fillStyle = inView ? ct.wallDark : darken(ct.wallDark, ct.revealedDarkness);

        case 'bricks':
          ctx.fillStyle = ct.wallLight;

        case 'cave':
```

```javascript
  const ct = CanvasTheme;
  const actualCount = Math.floor(count * ct.particleCountMult);
  const actualColor = ct.particleOverride || color;

  for (let i = 0; i < actualCount; i++) {
    const angle = Math.random() * Math.PI * 2;
    const speed = 1 + Math.random() * 3;
    combatParticles.push({

      vy: Math.sin(angle) * speed,
      life: 1,
      decay: ct.particleDecay,
      size: 2 + Math.random() * 3,
      color: actualColor,
      style: ct.particleStyle,      // circle | square | star | pixel
      trail: ct.particleTrail,
    });
  }
  // ...
}

// 渲染时根据 style 绘制不同形状
function drawParticle(ctx, p) {
  ctx.globalAlpha = p.life;
  ctx.fillStyle = p.color;
  const s = p.size * p.life;

  switch (p.style) {
    case 'circle':
      ctx.beginPath();
      ctx.arc(p.x, p.y, s, 0, Math.PI * 2);
      ctx.fill();
      break;
    case 'square':
      ctx.fillRect(p.x - s, p.y - s, s * 2, s * 2);
      break;
    case 'star':
      drawStar(ctx, p.x, p.y, s);
      break;
    case 'pixel':
      ctx.fillRect(Math.floor(p.x), Math.floor(p.y), 1, 1);
      break;
  }
}
```

### 6.3 性能影响分析

| 项目 | 影响 | 说明 |
|------|------|------|
| 主题数据内存 | ~3-4 KB | 6 个主题对象常驻内存，可忽略 |
| CSS 变量切换 | ~1ms | `setProperty` 批量写入，不在渲染热路径 |
| Canvas 读取 | 0 额外开销 | `CanvasTheme` 是普通对象，读取速度 = 硬编码常量 |
| 粒子数量增加 | +0-100% | `particleCountMult` 最高 2.0（虚空核心），低端机需注意 |
| 环境粒子 | +3-8 个 | 地城层叠加后每帧多画 3-8 个小矩形，可忽略 |
| 缩略图渲染 | ~5ms/次 | 仅在商城打开时渲染 6 次（80x60 小 Canvas），可接受 |
| **总体** | **可忽略** | 60fps 目标不受影响 |

**性能底线**：
- 低端设备降级策略：如果 `particleCountMult > 1.0` 且帧率 < 30fps，自动降至 1.0
- `glowBlur > 15` 时可降级为 10（`shadowBlur` 在部分浏览器较耗资源）
