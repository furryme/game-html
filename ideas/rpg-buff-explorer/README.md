# RPG Buff 探险 — 设计文档

## 项目概要

**单文件 HTML 像素风地城探险 Roguelite RPG**。无构建工具，无后端，localStorage 存档。

- **目标文件**：`web/games/dev-1-rpg.html`（已有基础版本，本设计为全新改版）
- **启动时间**：2026-06-19
- **灵感链**：TinyWind 像素航海调研 → 提炼 4 个通用框架 → 用 RPG 项目同时验证
- **设计方法**：多专家协作（共 28 人次），每份文档经独立设计 + Opus 整合
- **总文档量**：7 份，~4900 行

**核心循环**：
探索地城 → 踩到敌人进入回合制战斗 → 获得装备/金币/EXP → Boss 规则战 → 层前 buff 3 选 1 → 下一层 → 通关/死亡 → 永久成长

**MVP 范围**：
1 职业（战士），等级 1-10（每局从 1 级开始），3 层地城，10 技能，24 buff，3 栏位×3 稀有度装备，4 种敌人，3 个 Boss。

## 核心设计决策（MVP 裁剪后）

### 技术架构

- **单文件 HTML**，目标 gzip ~15KB
- **渲染**：Canvas + HTML 混合架构（Canvas 画动态内容，HTML 做 UI 交互），详见 [html加canvas混合架构](../html加canvas混合架构.md)
- **像素风**：480×270 分辨率，PIXEL=4，二维数组精灵，详见 [像素风程序化实现](../像素风程序化实现.md)
- **主题系统**：6 模块分层（palette/sprites/tiles/ui/effects/audio），继承 BASE_THEME 覆盖差异，支持混装（跨主题混搭模块）和运行时切换。5 个主题：经典/樱花/血月/星辰/虚空
- **UI 主题化**：`ThemeManager.switch()` 注入 CSS Custom Properties，Canvas 读取 `CanvasTheme` 缓存
- **buff 框架**：数据驱动，纯声明式数据 + `recalcBuffStats()` 单次聚合，战斗 O(1) 查询，详见 [buff选择框架](../buff选择框架.md)

### 成长系统

- **职业**：MVP 仅战士，后续扩展 6 专精（狂战士/骑士/圣骑士/元素师/奥术师/刺客/猎人）
- **等级**：1-10，每局从 1 级开始（Roguelite），通关获得灵魂碎片升级永久天赋
- **属性**：HP/ATK/DEF/SPD/CRIT/LUCK
- **伤害公式**：`max(1, ATK × 技能倍率 × buff倍率 - DEF × DEF/(DEF+100))`，减伤上限 80%
- **先手概率**：`我方SPD / (我方SPD + 敌方SPD)`
- **装备**：3 栏位（武器/护甲/饰品）× 3 稀有度（白/蓝/紫）× 随机词缀。层数倍率每层 +20%
- **技能**：10 个平铺"纹章"：斩击/旋风斩/防御姿态/战吼/重劈/反击/破甲/治疗术/烈焰斩/狂暴
- **buff**：层前 3 选 1，每局最多 3 个，24 buff 池，8 组组合效果
- **死亡惩罚**：保留等级/技能/buff 解锁 + 1 件遗物 + 50% 金币；丢失本局装备/消耗品
- **永久成长**：灵魂碎片 → 5 个线性天赋

### 战斗系统

- **回合制流程**：回合开始（DOT/HOT tick）→ 先手判定 → 先手行动 → 结算 → 后手行动 → 结算 → 下一回合
- **行动类型**：普通攻击/技能/防御姿态/物品/换装（战斗中可换装）
- **战斗状态**：中毒/灼烧/破防/硬直/增益/易伤（6 种）
- **Boss = 规则叠加物**（核心理念）：Boss 不是"更大血条"而是修改游戏规则（禁用技能/反转控制/HP 隐藏/随机 debuff 等），10 个可复用规则修改器。3 个 Boss：青苔守卫/骨骸领主/虚空领主
- **敌人 AI**：加权随机决策框架，4 种行为模板
- **回合节奏**：1000ms/1600ms/700ms 三档动画时长按行动类型区分

### 地城系统

- **3 层主题**：青苔回廊（绿调/5-7 房间/视野 4 格）→ 骨骸 Crypt（白灰调/5-7 房间/视野 3 格）→ 虚空核心（紫调/6-8 房间/视野 3 格 + 最终 Boss）
- **生成算法**：BLOB 膨胀（有机洞穴形状）+ MST 走廊（全联通）+ 20% 冗余环路
- **房间类型**：战斗/宝箱/陷阱/休息/神龛/boss（6 种）
- **事件库**：13 个随机事件（战斗/交易/道德抉择/纯收益/纯风险）
- **NPC**：商人/铁匠/治愈者/预言家（4 个）
- **雾战**：FOV 射线 180 条（2°步长），已探索/当前/未知三态
- **地图**：40×30 格，L 形走廊
- **货币**：金币 + 宝石（仅 2 种，宝石由 Boss 掉落）
- **战斗结果影响地城**：使用火焰技能 → 房间着火封锁路；治疗 → 出现花丛隐藏宝箱

### buff 联动

- **五维联动**：buff × 属性（6 个）/ buff × 技能（5 个）/ buff × 装备（4 个）/ buff × buff 组合（8 组）/ buff × Boss
- **组合示例**：火 + 风 = 烈焰风暴，盾 + 死 = 反弹护盾
- **叠加规则**：倍率乘法、加值加法，各有封顶（减伤 80%、闪避 50%）
- **解锁曲线**：第 1 局 8 个可用，第 5 局 16 个，全解锁约 20 局
- **数据架构**：`BUFF_DEFS` 声明式数据 → `recalcBuffStats()` 聚合到 `player.buffStats` 缓存 → `SYNERGY_DEFS` 独立 8 组通过 tags 交叉匹配

### 视觉规范

- **20 色调色板**：全局统一色号（PAL 常量），所有精灵/瓦片/UI 从此表取色
- **精灵**：玩家 8×12（地图）/ 16×20（战斗），敌人 8×8，Boss 16×20，二维数组表示
- **瓦片**：8×8 像素（×4 显示），3 层主题变体，离屏 Canvas 缓存
- **特效**：6 级视觉反馈（攻击→暴击→防御→技能→重伤→HP 危险），方块粒子
- **渲染管线**：主 Canvas（地图/角色）+ 粒子 Canvas（可选），帧循环不操作 DOM
