## 1. 战斗核心循环

### 1.1 地城全流程

```
  进入地城
    ↓
  层前大厅 → buff 3 选 1（每层一次）
    ↓
  ┌─── 第 N 层循环 ───┐
  │                    │
  │  地图上移动探索     │ ← 玩家驱动（方向键/WASD）
  │    ↓               │
  │  踩到敌人格子       │
  │    ↓               │
  │  进入战斗 ◄────────┼── 核心循环
  │    ↓               │
  │  战斗结算           │
  │    ↓               │
  │  返回地图           │
  │    ↓               │
  │  推进到下一格子     │
  └────┬──┬────────────┘
       │  └── 继续探索
       ↓  找到楼梯
  层前 buff 选择 → 下一层
```

**一层典型节奏**：战斗 → 探索/拾取 → 战斗 → 宝箱 → 战斗(Boss) → buff 3 选 1 → 下一层

### 1.2 战斗触发方式

| 触发方式 | 说明 |
|----------|------|
| 地图移动踩到敌人 | 走到敌人所在格子，立即进入战斗 |
| Boss 房间 | 每层最后一个房间，地图上以特殊标记显示 |

### 1.3 单场战斗——一回合完整流程

```
┌─── ① 回合开始阶段 ───
│  · 双方 DOT / HOT tick（中毒/灼烧等按回合扣血）
│  · 持续性 buff / debuff 回合计数 -1
│  · 到期效果自动移除
│
┌─── ② 先手判定 ───
│  先手概率 = 我方SPD / (我方SPD + 敌方SPD)
│  首次回合掷骰，此后先手顺序固定（除非被技能改变）
│
┌─── ③ 行动阶段（先手方）──
│  从可用行动中选一个执行：
│    · 普通攻击 / 技能 / 防御姿态 / 使用道具
│  点击后按钮立即禁用，播放行动动画
│
┌─── ④ 结算阶段（先手方）──
│  · 计算伤害 / 施加效果 / 显示飘字和粒子
│  · 目标死亡？ → 跳过对方行动 → 战斗胜利
│
┌─── ⑤ 行动阶段（后手方）──
│  · 显示"敌人准备攻击"预告（100ms）
│  · 按 AI 表或固定逻辑执行行动
│
┌─── ⑥ 结算阶段（后手方）──
│  · 计算伤害 / 施加效果
│  · 玩家 HP<=0？ → 战斗失败
│
└─── 回到 ① 回合开始阶段
```

### 1.4 行动点系统

MVP 采用**每回合 1 行动**的简化模式，不使用行动点数。

- 每回合先手方可执行 1 个行动
- 后手方可执行 1 个行动
- 部分技能可产生"追加效果"（如连斩 2 段伤害），但这属于技能自身效果，不额外消耗行动

后续迭代可引入行动点系统（AP），当前保持简单。

### 1.5 先手判定

```javascript
先手概率 = 我方SPD / (我方SPD + 敌方SPD)
```

- 仅在第一回合掷骰判定，后续回合保持同一先手顺序
- 若双方 SPD 相等，先手概率 = 50%
- Boss 战中先手概率额外 -10%（Boss 速度隐性加成）

## 2. 玩家行动

### 2.1 行动菜单

每回合玩家从以下行动中选择一项：

| 行动 | 消耗 | CD | 说明 |
|------|------|----|------|
| 普通攻击 | 无 | 无 | 基础攻击，ATK×1.0 |
| 技能（已解锁） | MP | 各技能不同 | 在快捷技能栏直接点击 |
| 防御姿态 | 无 | 无 | 本回合减伤 40%，回复 8% MP |
| 使用道具 | 对应道具×1 | 无 | 从背包选择消耗品 |

### 2.2 每回合行动次数

- 每回合 **1 次行动**
- 使用技能后不能再用普通攻击
- 使用物品后本回合结束
- 防御姿态后进入后手方行动

### 2.3 各行动消耗

| 行动 | MP 消耗 | CD（回合） | 备注 |
|------|---------|-----------|------|
| 普通攻击 | 0 | 无 | 始终可用 |
| 防御姿态 | 0 | 无 | 回复 8% MP |
| 使用道具 | 0 | 无 | 消耗背包物品 |
| 技能 | 见第 3 节技能表 | 见第 3 节 | — |

### 2.4 战斗中换装

MVP 阶段**不支持**战斗中换装。换装仅在探索阶段通过背包界面进行。

### 2.5 按钮禁用规则

- 玩家点击任意行动按钮后，**立即禁用所有按钮**（防止重复点击）
- 先手方动画播放完毕 → 显示敌人预告 → 敌人行动 → 全部结算完毕 → **恢复按钮**
- 任何动画播放期间按钮不可用

### 3.2 技能获取与解锁

- **默认技能**（1-4号）：开局即可用
- **等级解锁**（5-10号）：角色达到对应等级后自动解锁
- 每局游戏从 1 级开始（Roguelite），技能解锁进度每局重置
- 死亡后保留等级（元进度），下一局开局即拥有上次达到的等级对应的技能

### 3.3 技能释放节奏

```
玩家点击技能按钮
  ↓
检查 MP 是否足够 → 不足则弹窗提示
  ↓
检查 CD 是否结束 → 未结束则按钮灰显不可点
  ↓
扣除 MP，设置 CD 倒计时
  ↓
播放技能动画（600ms）
  ↓
执行技能效果（伤害计算 / 增益施加 / 治疗等）
  ↓
显示飘字 + 粒子
  ↓
进入后手方行动
```

### 3.4 CD 规则

- CD 以"回合"为单位
- 释放技能后 CD 设为对应数值
- 每回合开始时（阶段①）所有 CD -1
- **防御姿态不减少 CD**（防止刷防御等冷却）— CD 仅在正常回合流转时减少
- CD 归零后技能按钮恢复可用

### 3.5 快捷技能栏 UI

```
行动栏：
[⚔️ 攻击] [🛡️ 防御] [🎒 道具] [🏃 逃跑]

快捷技能（已解锁且可用的技能直接显示）：
[💥重击 8MP] [🗡️连斩 12MP 2回合] [📯战吼 6MP]
                                    [🌀旋风斩 15MP 🔒]  ← 冷却中灰显
```

- 快捷技能直接出现在行动栏下方，无需打开弹窗
- MP 不足时按钮灰显
- 冷却中的按钮显示倒计时并灰显
- 未解锁技能不显示

## 4. 战斗状态系统

### 4.1 状态总览

| 状态名 | 类型 | 施加方式 | 持续 | 效果 | 解除 |
|--------|------|----------|------|------|------|
| 中毒 | DOT | 敌人技能 / 环境 | N 回合 | 每回合开始损失 4% 最大 HP | 时间到期 / 解毒药 / 万能药 |
| 灼烧 | DOT | 敌方 Boss 规则 | N 回合 | 每回合开始损失固定伤害 | 时间到期 |
| 破防 | Debuff | 破甲技能 | 2 回合 | 目标 DEF -30%（可叠加，上限 -50%） | 时间到期 / 战斗结束 |
| 硬直 | Stun | 敌人重击 | 1 回合 | 目标跳过下一回合行动 | 时间到期（跳过后自动移除） |
| 增益·ATK | Buff | 战吼 / 战狂模式 | N 回合 | ATK × 倍率 | 时间到期 / 战斗结束 |
| 易伤 | Debuff | 暴击附加 / 特定技能 | N 回合 | 受到伤害 +10%/层（上限 +30%） | 时间到期 / 战斗结束 |

### 4.2 状态施加规则

```javascript
function applyStatus(target, status) {
  const existing = target.statuses.find(s => s.id === status.id);
  if (existing) {
    // 同类型状态：取持续时间更长的那个
    if (status.duration > existing.duration) {
      existing.duration = status.duration;
      existing.value = status.value;  // 更新效果值
    }
  } else {
    target.statuses.push({ ...status, turnsLeft: status.duration });
  }
}
```

- **DOT 类**（中毒/灼烧）：同名叠加时取持续时间更长者
- **Debuff 类**（破防/易伤）：同名叠加时取更强效果，易伤按层累计
- **Buff 类**（ATK 增益）：同名叠加时取更高采倍率
- **Stun 类**（硬直）：不可叠加，已眩晕时不再次眩晕

### 4.3 状态 tick 时机

所有持续状态在**回合开始阶段（①）**统一结算：

```
回合开始
  ↓
先处理玩家 DOT → 扣血 → 检查死亡
  ↓
再处理敌人 DOT → 扣血 → 检查死亡
  ↓
所有 buff/debuff 回合计数 -1
  ↓
移除到期状态
  ↓
进入先手判定
```

### 4.4 暴击与易伤的联动

- 玩家暴击时，对敌人额外施加 1 层"易伤"（受到伤害 +10%）
- 易伤最多叠加 3 层（+30%）
- 易伤持续时间 2 回合
- 这使得暴击build 不仅有单发高伤，还有后续 DOT 效果

## 5. 敌人 AI 框架

### 5.1 AI 决策模式

数据驱动的**加权随机行动表**。每个敌人类型定义一组可用行动，每场战斗独立初始化。

```
敌人回合
  ↓
读取该敌人的 actionTable
  ↓
根据当前 HP 百分比调整 weight（低血时攻击权重↑，防御权重↓）
  ↓
按 weight 加权随机选择一个行动
  ↓
执行行动
```

**低 HP 行为修正**：
- 敌人 HP < 30% 时，攻击类 weight × 1.5，防御类 weight × 0.3
- 敌人 HP < 15% 时，攻击类 weight × 2.0，增益类 weight × 0

### 5.3 难度缩放

敌人属性按地城层数缩放：

```
敌人HP  = 基础值 × (1 + 0.2 × (层数 - 1))
敌人ATK = 基础值 × (1 + 0.25 × (层数 - 1))
敌人DEF = 基础值 × (1 + 0.15 × (层数 - 1))
敌人EXP = 基础值 × (1 + 0.5 × (层数 - 1))
敌人金币= 基础值 × (1 + 0.3 × (层数 - 1))
```

| 层数 | HP 倍率 | ATK 倍率 | DEF 倍率 | EXP 倍率 |
|------|---------|----------|----------|----------|
| 1 | 1.0x | 1.0x | 1.0x | 1.0x |
| 2 | 1.2x | 1.25x | 1.15x | 1.5x |
| 3 | 1.4x | 1.5x | 1.3x | 2.0x |

## 6. Boss 战设计

### 6.1 设计哲学

**Boss = 规则叠加物**。Boss 不只是大血条，而是通过修改战斗规则制造压迫感。每个 Boss 携带 1-2 条"场规"，以标签形式在 UI 顶部常驻显示。

### 6.2 Boss 1：暗影法师（第 2 层 Boss）

| 项目 | 内容 |
|------|------|
| 名字 | 暗影法师 |
| 图标 | 🧙‍♂️ |
| 位置 | 第 2 层末尾 |
| HP | 150（经层数缩放） |
| ATK | 15 | DEF | 8 | SPD | 12 |

**场规——技能封印**：

| 效果 | 每回合开始时，随机禁用玩家一个非基础技能，持续到下一回合开始 |
|------|------|

```
玩家视角：
回合 1: [💥重击 🔒] [🗡️连斩] [📯战吼] [🛡️防御]
回合 2: [💥重击] [🗡️连斩 🔒] [📯战吼] [🛡️防御]
对策：普通攻击永远可用，靠普通攻击 + 防御拖过
```

**阶段系统**：

| 阶段 | HP 阈值 | 新增规则 |
|------|---------|----------|
| Ⅰ | 100%-50% | 技能封印（每次封印 1 个技能） |
| Ⅱ | <50% | 技能封印 + 暗伤（每回合对玩家造成固定 8 点暗伤） |

**掉落**：
- EXP: 80
- 金币: 40-60
- 装备掉落率: 100%（蓝色及以上稀有度）
- 灵魂碎片: +1

### 6.3 Boss 2（最终 Boss）：贪婪之王（第 3 层 Boss）

| 项目 | 内容 |
|------|------|
| 名字 | 贪婪之王 |
| 图标 | 👑 |
| 位置 | 第 3 层末尾（最终 Boss） |
| HP | 250（经层数缩放） |
| ATK | 22 | DEF | 14 | SPD | 10 |

**场规——装备腐化**：

| 效果 | 每 2 回合，随机一件装备属性 -20%（可叠加，上限 -60%） |
|------|------|

**场规——金币诱惑**：

| 效果 | 每 3 回合有 20% 概率弹出"放弃战斗，保留当前金币"选项 |
|------|------|

```
对策：
· 带足够多 buff 补足装备衰减
· 爆发速攻，2-3 回合结束战斗
· 连斩 > 重击（总伤更快，减少回合数）
· 破甲提前削弱 Boss 防御
```

**三阶段系统**：

| 阶段 | HP 阈值 | 生效规则 |
|------|---------|----------|
| Ⅰ | 100%-66% | 装备腐化（每 2 回合触发） |
| Ⅱ | 66%-33% | 装备腐化 + 金币诱惑（每 3 回合 20% 概率） |
| Ⅲ | <33% | 装备腐化 + 金币诱惑 + 狂暴（ATK +50%，攻击间隔缩短） |

**狂暴阶段额外效果**：
- 攻击从每回合 1 次变为有 30% 概率追加 1 次普通攻击
- UI 背景泛红，按钮闪烁频率加快

**掉落**：
- EXP: 200
- 金币: 80-120
- 装备掉落率: 100%（紫色稀有度）
- 灵魂碎片: +2
- 通关地城 → 进入结算画面

### 6.4 Boss 规则修改库（可复用）

以下规则可作为未来 Boss 的组合模块：

| # | 规则名 | ID | 效果 | 适用场景 |

### 7.3 回合节奏

```
普通攻击回合（总 ~1000ms）：
│── 玩家动画 300ms ──│── 空档 200ms ──│── 敌人预告 100ms ──│── 敌人攻击 400ms ──│

技能释放回合（总 ~1600ms）：
│── 技能动画 600ms ──│── 空档 300ms ──│── 敌人预告 200ms ──│── 敌人攻击 500ms ──│

防御回合（总 ~700ms）：
│── 防御姿态 200ms ──│── 敌人预告 100ms ──│── 敌人攻击(减半) 400ms ──│
```

**设计原则**：
- 普通攻击快，鼓励快速决策

- 防御回合更短（玩家主动选择"挨打跳过"）
- Boss 战整体更慢，给玩家更多决策时间

### 8.4 伤害计算公式

```javascript
// 基础伤害
function calcDamage(attackerAtk, skillMult, buffMult, targetDef) {
  // 减伤 = DEF / (DEF + 100)，上限 80%
  const mitigation = Math.min(0.8, targetDef / (targetDef + 100));

  // 装备腐化修正
  const corrosionMult = getCorrosionMultiplier(attackerEquiment);

  const rawDamage = attackerAtk * skillMult * buffMult * corrosionMult;
  const finalDamage = Math.max(1, Math.floor(rawDamage * (1 - mitigation)));

  // 暴击判定
  const isCrit = Math.random() * 100 < attackerCrit;
  if (isCrit) {
    return { damage: Math.floor(finalDamage * 1.5), isCrit: true };
  }
  return { damage: finalDamage, isCrit: false };
}
```

**关键参数**：
- 减伤公式：`DEF / (DEF + 100)`，上限 80%
- 暴击倍率：1.5x
- 防御姿态额外减伤：40%（与公式减伤叠加，总上限 95%）
- 最小伤害：1
