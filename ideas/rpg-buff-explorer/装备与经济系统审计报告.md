## 4. 装备获取流程分析

### 4.1 设计意图

```
敌人战败 (18%概率)  -->  装备掉落  -->  玩家拾取
宝箱开启 (未识别)    -->  装备获得  -->  鉴定后使用
Boss击败 (100%)     -->  稀有+装备  -->  直接获得
                     深层诅咒宝箱   -->  强装备+负面词缀
```

### 4.2 实际代码流程

```
商店刷新 (35%概率含装备)  -->  金币购买  -->  加入背包
                               |
上层强化装备 (3宝石)    -->  直接修改已装备武器的 stats
                               |
遗物继承 (permanant.equipRelic) -->  新局开始时自动装备
                               |
[敌人战败掉落]  -->  rollLoot() 存在但从未调用  -->  XXX 断裂
[宝箱装备]      -->  未实现                      -->  XXX 断裂
```

### 4.3 关键问题

1. **`rollLoot()` 函数孤立** — `equip.js:161-173` 定义了装备掉落逻辑（18%基础概率+每层3%），但整个代码库中没有调用它的代码。敌人战败 `enemyDefeated()` 只发放 EXP 和金币。

2. **唯一装备来源是商店** — 商店只有35%概率包含1-2件装备，且需要金币购买。早期玩家（20金起步）很难买到第一件武器。

3. **没有装备分解系统** — 设计中的"重复装备自动分解为强化石"不存在，所以也不会产生强化石。

4. **Boss必定掉落装备的设计未实现** — `enemyDefeated()` 中没有装备掉落逻辑，Boss被击败也不掉落装备。

5. **诅咒装备未实现** — 设计中3层的诅咒宝箱（强装备+负面词缀）不存在。

## 5. 宝石经济分析

### 5.1 设计意图

```
Boss击败 --> 固定3宝石
深层宝箱 --> 极低概率1宝石
预计每局: 3-4颗宝石

消费:
  武器强化 +2ATK --> 3宝石 (递增: 3,4,5...)
  防具强化 +2DEF --> 3宝石 (递增: 3,4,5...)
预期: 每局约1次强化
```

### 5.2 实际代码


### 5.3 结论

**宝石系统处于完全断裂状态：**
- 玩家无法通过任何路径获得宝石
- 宝石消费UI存在（`showGemEnhancement()` 在 buff 选择后自动触发如果 gems > 0）

- 但因为没有宝石来源，这些消费路径实际上不可达
- 死亡处理也不正确：`onDeath()` 不处理宝石保留

## 7. 修复建议 (按优先级排序)

### P0 — Critical (阻断核心体验)

#### 7.1 接入宝石来源

**问题**: 玩家无法获得宝石，宝石消费UI和强化功能完全不可达。

**修复方案**:
1. 在 `nextFloor()` 流程中（`movement.js:262-315`）接入 `collectFloorReward()`，使层完成奖励发放
2. 在 `onVictory()` 流程中（`persistence.js:300-317`）接入 `calculateClearReward()`
3. 或者更简单：在 `enemyDefeated()` 的 Boss 分支中直接添加宝石掉落（符合设计文档"Boss掉落3颗"）

**推荐**: 方案3优先（最快见效），方案1+2 后续补充

```
// combat.js enemyDefeated() 的 Boss 分支中添加:
if (enemy.isBoss) {
  player.gems += 3;  // Boss固定掉落3颗宝石
  // ... 现有逻辑
}
```

#### 7.2 修复宝石死亡处理

**问题**: `onDeath()` (persistence.js) 不处理宝石，死亡后宝石全部丢失；`onGameOver()` (economy.js) 有50%保留逻辑但从未调用。

**修复方案**: 在 `onDeath()` 中添加宝石处理，统一两个死亡处理器：

```
// persistence.js onDeath() 中添加:
const keepGems = Math.floor(player.gems * 0.5);
permanent.gems = (permanent.gems || 0) + keepGems;
```

#### 7.3 接入装备掉落

**问题**: `rollLoot()` 存在但无人调用，敌人战败不掉落装备，Boss也不掉落装备。唯一来源是商店。

**修复方案**: 在 `enemyDefeated()` (combat.js) 中调用 `rollLoot()`，将生成的装备添加到地图或背包：

```
// combat.js enemyDefeated() 末尾添加:
const loot = rollLoot(floorNum);
if (loot) {
  // 添加到背包或地图位置
  player.inventory.push(loot);
}
```

### P1 — High (显著影响游戏体验)

#### 7.4 接入层完成奖励

**问题**: `collectFloorReward()` 从未调用，玩家完成层后不获得金币奖励。

**修复方案**: 在 `nextFloor()` 开头或 `showFloorBreak()` 之前调用 `collectFloorReward()`。

#### 7.5 实现装备分解系统

**问题**: 设计中有"重复装备自动分解为强化石"，代码中不存在。

## 8. 经济循环对比

### 8.1 设计意图

```
战斗 --> 金币 + EXP + 装备掉落 + 宝石(Boss)
    |
    v
商店(金币) --> 药水/炸弹/钥匙/卷轴/鉴定
    |
    v
层间断: 宝石强化(武器+2ATK / 防具+2DEF)
    |
    v
Boss通关 --> 灵魂碎片 --> 永久天赋(元成长)
    |
    v
死亡 --> 保留1件遗物装备 + 1个遗物Buff --> 下一局更强
```

### 8.2 实际运行

```
战斗 --> 金币 + EXP + [装备掉落XXX] + [宝石XXX]
    |
    v
商店(35%概率有装备) --> 药水/卷轴/Occasionally装备
    |
    v
层间断: [宝石强化XXX — 无宝石] + Buff选择(3选1) + [宝石临时强化XXX]
    |
    v
Boss通关 --> 灵魂碎片+3 --> 永久天赋
    |
    v
死亡 --> 保留50%金币 + 遗物(Buff或装备) --> 下一局开始
```

**核心断裂点**: 装备掉落和宝石来源未接入，导致两条核心经济链断裂。
