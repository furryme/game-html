# 商店/商人系统设计

> 基于审计报告重新设计
> 目标：修复经济循环断裂，让金币有明确的消耗出口

---

## 一、现有设计评估

**现有设计（地城生成.md 第 5.7 节）方向正确但存在结构性缺陷。**

### 优点
- 商品清单完整（8 种消耗品），覆盖恢复/探索/战斗三大需求
- 价格公式设计合理（随层数递增）
- 两层商店（常驻 + 流浪）的构想符合地城探索节奏

### 致命缺陷
1. **商店房间类型缺失** — `assignRoomTypes()` 没有 `shop` 类型，`openShop()` 无人调用，代码写了但永远打不开
2. **流浪商人事件过于简陋** — 只有 1 种商品（买药水 15 金），无商品列表，无层数差异，无战略选择
3. **卖装备逻辑是负面体验** — 自动卸下第一个装备槽，玩家无法选择卖什么
4. **价格公式不一致** — 设计写 `x(1 + floorNum * 0.15)`，代码用 `x(1 + 0.3 * (floorNum - 1))`，后者第 3 层贵 60%（vs 设计 45%）
5. **经济循环断裂** — 金币来源充足（战斗 + 宝箱 + 事件）但消耗出口被阻塞，金币变成无用数字

### 结论：需要全新设计

保留原设计的商品池概念和两层商店（常驻 + 流浪）框架，彻底重写实现方案。

---

## 二、新设计方案

### 2.1 核心架构：两种商店

```
玩家获得金币
    |
    +---> [A] 常驻商店（每层 1 个房间）
    |       标准价格，全商品池，3-5 件随机商品
    |       可重复进入，每次进入刷新商品
    |
    +---> [B] 流浪商人（随机事件）
            限时特价，1-2 件商品，价格 7 折
            每层 0-1 次触发
```

### 2.2 商品列表

按实用性分 3 档，移动端 UI 每次展示 3-5 件。

#### 第一档：主力消耗品（每次必出 1-2 件）

| 商品 | 图标 | 基础价 | 战斗帮助 | 权重 |
|------|------|--------|---------|------|
| 生命药水 | (red_heart) | 25 | 回复 80 HP，核心恢复手段 | 30 |
| 魔力药水 | (blue_heart) | 20 | 回复 40 MP，释放技能必需 | 20 |
| 炸弹 | (bomb) | 35 | 战斗中 60 无视防御伤害，Boss 战神器 | 20 |
| 解毒药 | (test_tube) | 15 | 解除中毒（第 2 层+ 陷阱常见） | 15 |

#### 第二档：高级消耗品（高层概率出现）

| 商品 | 图标 | 基础价 | 战斗帮助 | 权重 | 最低层 |
|------|------|--------|---------|------|--------|
| 大生命药水 | (sparkling_heart) | 60 | 回复 200 HP，Boss 战前必备 | 10 | 1 |
| 万能药 | (sparkles) | 100 | 全回复 HP+MP，清异常，局内翻盘 | 5 | 2 |
| 探测卷轴 | (magnifying_glass) | 25 | 揭示当前房间所有陷阱 | 8 | 1 |
| 银钥匙 | (key) | 30 | 开上锁的门（当前无门，预留） | 5 | 1 |

#### 第三档：稀有物品（低概率，高价值）

| 商品 | 图标 | 基础价 | 战斗帮助 | 权重 | 最低层 |
|------|------|--------|---------|------|--------|
| 鉴定卷轴 | (scroll) | 50 | 鉴定未识别装备 | 8 | 1 |
| 传送卷轴 | (cyclone) | 80 | 传送到随机安全位置 | 5 | 2 |
| 随机装备 | -- | 见下方 | 直接提升属性 | 见下方 | 1 |

**装备作为商店商品**：
- 每轮有 30% 概率出 1 件装备（原设计 35%，降低以控制商品数）
- 装备价格 = `基础价(白 15 / 蓝 40 / 紫 80) + 属性总和 x 1.5`，再乘以层数倍率
- 不改变现有 `generateEquipment()` 逻辑，复用即可

#### 商品池总结

| 层数 | 消耗品数量 | 含装备概率 | 典型商品 |
|------|-----------|-----------|---------|
| 第 1 层 | 2-3 件 | 30% | 生命药水、魔力药水、炸弹、解毒药 |
| 第 2 层 | 3-4 件 | 30% | + 大生命药水、万能药、探测卷轴 |
| 第 3 层 | 3-4 件 | 30% | + 万能药、传送卷轴、鉴定卷轴 |

### 2.3 价格系统

#### 公式

```
商品价格 = Math.round(基础价 x 层数倍率)
层数倍率 = 1 + 0.15 x (floorNum - 1)
```

| 层数 | 倍率 | 生命药水(25) | 炸弹(35) | 万能药(100) |
|------|------|-------------|---------|------------|
| 1 | 1.00 | 25 | 35 | 100 |
| 2 | 1.15 | 29 | 40 | 115 |
| 3 | 1.30 | 33 | 46 | 130 |

**修正原因**：原代码倍率 `1 + 0.3*(floorNum-1)` 第 3 层 = 1.6，商品贵 60%，超过设计意图（45%）。改为 `1 + 0.15*(floorNum-1)` 第 3 层 = 1.3，贵 30%，更合理。

#### 流浪商人折扣

流浪商人的商品价格 = 标准价格 x 0.7（7 折）。

### 2.4 常驻商店（商店房间）

#### 房间生成

- 在 `assignRoomTypes()` 中加入 `shop` 类型
- 第 1 层保证 1 个商店房（如果中间房间数 >= 3）
- 第 2-3 层每个层必出 1 个商店房
- 商店房不参与 combat/loot/trap 等概率分配，优先分配

分配规则（修改 `assignRoomTypes`）：
```
rooms[0].type = 'empty'          // 起始
rooms[last].type = 'combat'      // Boss + 楼梯
// 中间房间中：
//   先确保 1 个 shop + 1 个 loot
//   其余按原概率分配 combat/loot/trap/resting/shrine/empty
```

#### 进入商店

玩家走到商店房间的中心格（cx, cy）：
1. 检测当前房间 type === 'shop'
2. 弹出商店 UI（调用已有的 `openShop(floorNum)`）
3. 游戏暂停（`gameState.paused = true`）

**可重复进入**：关闭商店 UI 后回到地图，再次走进同一个商店房间，重新弹出商店（商品刷新）。

#### 商品刷新规则

每次进入商店：
- 重新调用 `generateShopItems(floorNum)` 生成全新商品列表
- 同一局内商品池不变（不跨局累积）
- 不记录"已购买"状态（可以重复购买同种商品，只要买得起）

#### 商店 NPC 视觉

在地图上商店房间中心放置商人精灵：
- 图标：使用 emoji `(man_beard)` 绘制在 canvas 上
- 旁边放一个小摊位标记（深色矩形）
- 进入中心格触发，不需要额外交互按钮

### 2.5 流浪商人（随机事件）

替换现有的 `wandering_merchant` 事件。

#### 触发规则

- 保留现有触发机制：走廊格，每步 5% 概率，每层最多 2 个事件
- 从 14 个事件中随机选取，商人事件只是其中之一
- `minFloor: 1`（第 1 层起可触发）

#### 事件内容

流浪商人事件不再硬编码商品，而是**调用商店系统生成特价商品**。

事件弹窗 UI 流程：
1. 显示商人描述文字
2. 调用 `generateWanderingShopItems(floorNum)` 生成 2 件特价商品
3. 渲染商品列表（商品名、原价划线、折后价、购买按钮）
4. 玩家可以选择购买 0-N 件
5. 关闭事件，回到地图

`generateWanderingShopItems` 逻辑：
- 从 4 个主力消耗品中随机选 2 种（权重同 `SHOP_CONSUMABLE_KEYS` 但排除高级品）
- 价格 = 基础价 x 层数倍率 x 0.7
- 不出售装备（保持事件轻量）

#### 具体实现

修改 `events.js` 中 `wandering_merchant` 定义：
```js
{
  id: 'wandering_merchant',
  title: '流浪商人',
  desc: '一个推着小推车的商人在走廊转角出现。"冒险者？今天特价。"',
  category: 'npc',
  minFloor: 1,
  // 不再用 choiceA/choiceB，改用特殊 effect
  effect: 'wandering_shop',
}
```

修改 `triggerEvent()`：当 `event.effect === 'wandering_shop'` 时，调用 `openWanderingShop(floorNum)` 而非渲染 choice 按钮。

修改 `event.js` 的 `applyEventEffect`：移除 `buyPotion` 和 `sellEquipment` 两个 effect handler（不再需要）。

### 2.6 经济平衡

#### 金币收入估算（每层）

| 来源 | 第 1 层 | 第 2 层 | 第 3 层 | 备注 |
|------|---------|---------|---------|------|
| 初始金币 | 20 | -- | -- | 开局固定 |
| 小怪掉落 (6-8 场) | 30-60 | 45-90 | 60-120 | 每怪 5-15 金 x 层数倍率 |
| Boss 掉落 | 80-120 | 80-120 | 80-120 | 固定 |
| loot 房间 | 10-30 | 15-35 | 20-40 | 每层 1-2 个 |
| 事件奖励 | 0-40 | 0-40 | 0-40 | 看脸 |
| **小计** | **140-250** | **140-287** | **160-320** | 含初始金 |

#### 合理支出（一局通关）

| 消费 | 预计花费 | 说明 |
|------|---------|------|
| 生命药水 x4 | 100 (层 1 价) | 核心恢复，约每层 1-2 瓶 |
| 炸弹 x2 | 70 (层 1 价) | Boss 战用 |
| 魔力药水 x2 | 40 | 技能释放 |
| 解毒药 x1 | 15 | 第 2 层+ |
| 探测卷轴 x1 | 25 | 陷阱房 |
| **合理消费总计** | **~250 金** | 约占收入 70-80% |

**平衡目标**：玩家通关一局，金币基本花完（留 20-50 金），死亡后保留 50%。

#### 关键验证

- 第 1 层打完：大约 80-150 金，可以买 2-3 个商品后进入第 2 层
- 第 2 层打完：大约 120-200 金累积，可以买高级商品
- 第 3 层打完：如果前面没花光，通关时带几十金死亡

修改：保证至少 1 个 shop（如果中间房间 >= 3）：
```js
function assignRoomTypes(rooms, floorNum) {
  rooms[0].type = 'empty';
  rooms[rooms.length - 1].type = 'combat';
  var middle = rooms.slice(1, -1);
  shuffle(middle);

  // 确保至少 1 个 loot 和 1 个 shop（如果房间够）
  var assigned = 0;
  if (middle.length > 0) { middle[assigned].type = 'loot'; assigned++; }
  if (middle.length >= 3) { middle[assigned].type = 'shop'; assigned++; }

  for (var i = assigned; i < middle.length; i++) {
    var roll = Math.random() * 100;
    if (roll < 45) middle[i].type = 'combat';
    else if (roll < 60) middle[i].type = 'loot';
    else if (roll < 72) middle[i].type = 'trap';
    else if (roll < 82) middle[i].type = 'resting';
    else if (roll < 90) middle[i].type = 'shrine';
    else middle[i].type = 'empty';
  }
}
```

```js
function generateWanderingShopItems(floorNum) {
  // 从主力消耗品中选 2 件，价格打 7 折
  var coreKeys = SHOP_CONSUMABLE_KEYS.filter(function (k) {
    return k.minFloor <= floorNum && k.weight >= 15;
  });
  var items = [];
  var used = {};
  for (var i = 0; i < 2; i++) {
    var key = null;
    for (var a = 0; a < 20; a++) {
      var pick = coreKeys[Math.floor(Math.random() * coreKeys.length)];
      if (!used[pick.key]) { key = pick; break; }
    }
    if (!key) break;
    used[key.key] = true;
    var def = ITEMS_DATA[key.key];
    var price = Math.round(def.price * shopPriceMultiplier(floorNum) * 0.7);
    items.push({
      id: 'wand_' + i,
      itemId: key.key,
      price: price,
      origPrice: Math.round(def.price * shopPriceMultiplier(floorNum)),
      label: def.name,
      icon: def.icon,
      type: 'consumable',
    });
  }
  return items;
}
```

6. 离开商店后，重新走进同一商店，商品刷新
7. 流浪商人事件触发时，显示 2 件特价商品
8. 第 2 层商店可能出现万能药和传送卷轴
9. 第 1 层商店不出现万能药和传送卷轴

### 6.2 经济验证

1. 打完全局 3 层，累计收入约 200-400 金
2. 在商店消费 150-300 金后仍有余额
3. 商品价格第 3 层比第 1 层贵 30%（不是 60%）
4. 流浪商人价格比常驻商店便宜 30%
