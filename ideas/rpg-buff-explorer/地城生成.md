# 地城探险 — 整合设计文档

> 单文件 HTML 像素风 RPG，地城探险
> MVP：1 职业（战士），等级 1-10，3 层地城
> 每层 4-7 个房间 + 1 个 boss 房间（第 3 层末端）
> 死亡后保留等级/技能/buff 解锁，丢失本局装备/金币（Roguelite）

---

## 1. 地城结构

### 1.1 层数总览

| 层数 | 名称 | 房间数 | 房间类型数 | 敌人数量 | 陷阱密度 | 视野半径 |
|------|------|--------|-----------|---------|---------|---------|
| 1 | 青苔回廊 | 5-7 | 含 1 商店/1 宝箱 | 6-8 | 0% | 4 格 |
| 2 | 骨骸 Crypt | 5-7 | 含 1 神龛/1 宝箱 | 8-10 | 8%（走廊） | 3 格 |
| 3 | 虚空核心 | 6-8 | 含 1 神龛/1 宝箱 + Boss | 10-12 | 12%（走廊） | 3 格 |

### 1.2 房间连接方式

- **生成阶段**：所有房间通过最小生成树（MST）走廊连接，保证全联通
- **冗余走廊**：20% 概率在任意两个未直连房间之间追加一条 L 形走廊，形成环路
- **可回溯**：玩家可以原路返回，没有单向门，鼓励探索全图后找楼梯

### 1.3 路径设计

```
入口房间(起始, empty)
  → 若干战斗/宝箱/陷阱/休息房间（网状分支）
  → 最后一个房间 = 楼梯（前两层）/ Boss + 楼梯（第三层）
```

- 起始房间固定为第一个生成的房间，内含玩家出生点，无敌人
- 楼梯固定在最后一个生成的房间中心
- 第 3 层楼梯前必须击杀 Boss
- 玩家通过小地图 + 战争迷雾判断楼梯位置（已揭露的楼梯格显示金色标记）

### 1.4 楼层切换流程

```
进入楼层 → 显示楼层横幅（名称 + 环境 debuff 描述，1.2s）
  → 玩家出生在起始房间中心
  → 探索 → 战斗 → 拾取
  → 到达楼梯 → 进入 buff 三选一
  → 生成下一层 → 重复
  → 第 3 层击杀 Boss 后 → 胜利结算
```

## 2. 地图生成算法

### 2.1 算法选择：BLOB 膨胀 + MST 走廊

不用 BSP（太规整），不用纯随机游走（太迷宫化）。

**BLOB 膨胀法**：在墙壁地图上撒种子点，每颗种子向外膨胀随机次数，形成有机洞穴形状。

### 2.2 参数表

| 参数 | 值 | 说明 |
|------|------|------|
| 地图尺寸 | 40×30 格 | 格子单位，桌面端 16px/格 |
| 种子数（房间目标） | 5-8 | 随层数递增 |
| 种子最小间距 | 4 格 | 防止种子重叠 |
| 每种子膨胀次数 | 2-4 次（随机） | 控制房间大小 |
| 最大膨胀总轮数 | 6 | 防止房间过大 |
| 走廊宽度 | 第 1 层 2 格 / 其余 1 格 | 视觉区分 |
| 冗余走廊概率 | 20% | 形成环路 |
| 走廊样式 | L 形（先横后纵 or 先纵后横，随机） | 宽度 1-2 格 |

### 2.3 生成流程伪代码

```
generateFloor(floorNum):
  theme = FLOOR_THEMES[floorNum]
  roomTarget = rng(theme.roomTarget[0], theme.roomTarget[1])

  // ① 全地图填 WALL
  grid = fillWalls(MAP_W, MAP_H)

  // ② 撒种子
  seeds = []
  for i = 0 to roomTarget * 3:
    sx = rng(2, MAP_W - 3)
    sy = rng(2, MAP_H - 3)
    if seeds 中存在曼哈顿距离 < 4 的: continue
    seeds.push({ x: sx, y: sy, frontier: [{x, y}] })
    if seeds.length >= roomTarget: break

  // ③ 膨胀
  growTimes = seeds.map(() => rng(2, 4))
  for pass = 0 to 5:
    for i = 0 to seeds.length:
      if pass >= growTimes[i]: continue
      newFrontier = []
      for f in seeds[i].frontier:
        for [dx, dy] in DIRS4:
          nx = f.x + dx, ny = f.y + dy
          if inBounds(nx, ny) AND grid[ny][nx] == WALL:
            grid[ny][nx] = FLOOR
            newFrontier.push({x: nx, y: ny})
      seeds[i].frontier.push(...newFrontier)

  // ④ 从膨胀轮廓提取房间 AABB
  rooms = seeds.map(s => aabbFromCells(grid, s.frontier))

  // ⑤ MST 走廊连接
  edges = all pairs of rooms, sorted by manhattan distance
  unionFind = init(rooms.length)
  for edge in edges:
    if find(edge.a) != find(edge.b):
      union(edge.a, edge.b)
      carveCorridor(grid, rooms[edge.a], rooms[edge.b], theme.corridorWidth)

  // ⑥ 冗余走廊
  if random() < 0.2:
    i = rng(0, rooms.length-1), j = rng(0, rooms.length-2)
    if j >= j: j++
    carveCorridor(grid, rooms[i], rooms[j], theme.corridorWidth)

  // ⑦ 分配房间类型
  types = assignRoomTypes(rooms, floorNum)

  // ⑧ 放置内容
  playerStart = rooms[0].center
  stairsPos   = rooms[rooms.length-1].center
  placeEnemies(grid, rooms, types, theme, playerStart)
  placeItems(rooms, types, floorNum)
  placeTraps(grid, theme, playerStart)

  // ⑨ 初始化战争迷雾
  revealed = all false
  revealLineOfSight(grid, playerStart, theme.visibility)

  return { grid, rooms, types, enemies, items, traps,
           playerStart, stairsPos, theme, revealed, visibility: theme.visibility }
```

### 2.4 走廊雕刻

```
carveCorridor(grid, roomA, roomB, width):
  x = roomA.cx, y = roomA.cy
  // 随机先横 or 先纵
  if random() < 0.5:
    while x != roomB.cx: grid[y][x] = CORRIDOR; x += sign(roomB.cx - x)
    while y != roomB.cy: grid[y][x] = CORRIDOR; y += sign(roomB.cy - y)
  else:
    while y != roomB.cy: grid[y][x] = CORRIDOR; y += sign(roomB.cy - y)
    while x != roomB.cx: grid[y][x] = CORRIDOR; x += sign(roomB.cx - x)

  if width == 2:
    // 在走廊旁再挖一行
    // （略，沿走廊方向偏移 1 格）
```

### 2.5 生成后校验

```
validateFloor(dungeon):
  if dungeon.rooms.length < 4: 重新生成
  if bfs(dungeon.playerStart) 不能到达 dungeon.stairsPos: 重新生成
  if dungeon.enemies.length == 0: 在随机房间补 1 个敌人
  // 检查是否存在 loot 房间
  if !dungeon.types.includes('loot'): 将随机 empty 房间改为 loot
```

## 3. 房间类型详表

### 3.1 类型分配规则

```
每层房间分配（除起始和末尾房间外）：

起始房间 = empty（玩家出生）
末尾房间 = combat（含楼梯，第 3 层含 Boss）
其余房间按以下概率分配：
  roll = random(0, 100)
  roll < 45  → combat  （至少出现 1 次）
  roll < 60  → loot    （至少出现 1 次）
  roll < 70  → trap
  roll < 80  → resting
  roll < 90  → shrine  （每层最多 1 个）
  else      → empty
```

### 3.2 房间类型详表

| 类型 | 概率 | 内容 | 视觉标记 |
|------|------|------|---------|
| **combat** | 45% | 1-2 个敌人（距离加权强度），随机 1 个药水 | 无特殊 |
| **loot** | 15% | 1 个宝箱 + 0-1 个道具 | 地板中央有微光 |
| **trap** | 10% | 2-3 个陷阱（含 1 个隐蔽）+ 1 个敌人 | 地面颜色略暗 |
| **resting** | 10% | 一个休息点（走到中心格回复 30% HP），无敌人 | 地板上有火焰图标 |
| **shrine** | 10% | 进入房间后弹出三选一 buff | 地板中央有光柱 |
| **empty** | 10% | 空房间，可能掉落 1 个小药水 | 无特殊 |

### 3.3 宝箱细分

| 宝箱等级 | 颜色 | 触发条件 | 内容 |
|---------|------|---------|------|
| 普通宝箱 | 棕色 | loot 房间默认 | 随机药水×2 或 金币 10-25 |
| 精良宝箱 | 蓝色 | 第 2 层以上 loot 房（50% 概率） | 该层可用装备（未鉴定）+ 金币 15-30 |
| 诅咒宝箱 | 暗红 | 第 3 层 loot 房（20% 概率） | 强力装备但有 1 条负面词条 |
| 陷阱宝箱 | 正常外观 | 10% 概率附加陷阱 | 开启时触发毒气/噬金/幻影（见第 5 节） |

## 5. 宝藏与掉落

### 5.1 货币体系

MVP 仅两种货币：

| 货币 | 获取途径 | 消耗途径 | 设计意图 |
|------|---------|---------|---------|
| 金币 | 怪物掉落、宝箱 | 商店买药水/道具 | 基础流通货币，量大 |
| 宝石 | Boss 掉落（2-4）、深层宝箱极低概率 | 层间强化（武器 +2 ATK 或 防具 +2 DEF，消耗 3 宝石） | 硬通货，驱动长期目标 |

**死亡结算**：金币清零，宝石清零。永久成长走 buff 解锁系统（localStorage）。

### 5.2 装备掉落表

装备掉落走权重表，不按层硬编码：

```
LOOT_TABLES = {
  common: { weight: 500, items: ["铁剑", "锁子甲", "布甲"] },
  rare:   { weight: 200, items: ["钢剑", "铁甲", "暗影匕首"] },
  epic:   { weight: 80,  items: ["烈火剑", "秘银甲", "圣剑"] },
}

rollLoot(floorNum):
  pool = all items where item.minFloor <= floorNum
  roll = weighted random by rarity weight
  return pool[roll]

装备属性 = 基础值 × (1 + (floorNum - item.minFloor) × 0.1)
```

**掉落概率**：
- 小怪：18% 概率掉落装备
- Boss：100% 掉落 1 件 epic 级以上装备 + 2-4 宝石
- 陷阱房敌人：25% 概率掉落

### 5.3 装备栏位

| 栏位 | 说明 | 示例 |
|------|------|------|
| 武器 | ATK / 可能附带 MP 或 暴击 | 铁剑(4ATK)、烈火剑(14ATK) |
| 防具 | DEF | 布甲(3DEF)、龙鳞甲(22DEF) |
| 饰品 | 特殊效果（第 2 阶段加入） | 力量戒指(ATK+2)、生命之戒(maxHP+15) |

MVP 先用 2 栏位（武器 + 防具），饰品在 P2 加入。

### 5.4 稀有度与鉴定

- **common（白）**：直接显示属性
- **rare（蓝）**：直接显示属性
- **epic（紫）**：未鉴定状态，只显示"史诗装备·2 条词条"，需用鉴定卷轴（商店 50 金）或层间免费鉴定
- **诅咒装备（暗红）**：1 条正面词条 + 1 条负面词条，直接显示

### 5.5 装备词条池

```
武器词条：
  ATK +N        (N = 3-20，随层数递增)
  MP +N         (N = 5-20)
  暴击率 +N%    (N = 5-20)

防具词条：
  DEF +N        (N = 2-22)
  maxHP +N      (N = 10-50)
  减伤 N%       (N = 5-15)

诅咒词条（负面）：
  DEF -N
  maxHP -N
  速度 -N
```

**触发逻辑**：

```
checkTrap(dungeon, pos):
  trap = dungeon.traps.find(t => t.x == pos.x && t.y == pos.y && !t.triggered)
  if !trap: return false
  if random() < trap.detectChance:
    log("你发现了" + trap.icon + "陷阱并绕开了")
    trap.revealed = true
    return true
  trap.triggered = true
  applyTrapEffect(trap, player)
  return true
```

## 10. 技术实现

### 10.1 瓦片系统

```
TILE = { WALL: 0, FLOOR: 1, CORRIDOR: 2 }
```

**渲染尺寸**：桌面端 16px/格，移动端 12px/格

**墙壁渲染**：填色矩形 + 顶部 1px 亮色横线（砖缝高光）
**地板渲染**：按主题色填色 + 棋盘格交替 + 主题纹理叠加

### 10.2 地图数据结构

```
Dungeon = {
  // 网格
  grid:       Number[MAP_H][MAP_W],    // TILE.WALL / FLOOR / CORRIDOR
  width:      Number,                  // 40
  height:     Number,                  // 30

  // 房间
  rooms:      [{ x, y, w, h, cx, cy, type }],
  types:      String[],                // room type per room index

  // 实体
  enemies:    [{ x, y, name, icon, hp, maxHp, atk, def, exp, gold, boss? }],
  items:      [{ x, y, type }],
  traps:      [{ x, y, type, triggered, revealed }],

  // 导航
  playerStart:{ x, y },
  stairsPos:  { x, y },

  // 楼层属性
  theme:      { id, name, colors, envBuff, enemyPool, trapChance, visibility },
  revealed:   Boolean[MAP_H][MAP_W],   // 战争迷雾
}
```

### 10.3 战争迷雾（Raycast 视线）

```
revealLineOfSight(grid, px, py, radius):
  for angle = 0 to 358, step 2:       // 180 条射线
    rad = angle × PI / 180
    dx = cos(rad), dy = sin(rad)
    x = px + 0.5, y = py + 0.5
    for step = 0 to radius - 1:
      tx = floor(x), ty = floor(y)
      if !inBounds(tx, ty): break
      revealed[ty][tx] = true
      if grid[ty][tx] == WALL AND random() > 0.3: break  // 墙阻挡
      x += dx * 0.5
      y += dy * 0.5
```

三种状态：
- `unseen`（纯黑）：从未被 raycast 覆盖
- `seen`（灰度）：曾被揭露但当前不在视野内
- `visible`（全彩）：当前 raycast 覆盖

### 10.4 碰撞检测

```
canMoveTo(nx, ny):
  if !inBounds(nx, ny): return false
  if grid[ny][nx] == TILE.WALL: return false
  return true
```

移动优先级：
1. 检查 wall 碰撞
2. 检查 enemy（进入战斗）
3. 检查 item（自动拾取）
4. 检查 trap（触发）
5. 检查 stairs（下楼）

### 10.5 相机系统

MVP 不需要相机——地图 40×30 × 16px = 640×480，适配 Canvas 画布居中显示。

移动端：Canvas 固定为屏幕宽度，地图滚动跟随玩家（viewport 跟随）。

### 10.6 渲染管线

```
render():
  // ① 离屏缓存静态层（地图 tile 变化时才重绘）
  if staticCacheDirty:
    renderStaticLayer(offscreenCanvas)

  // ② 绘制到主 Canvas
  drawImage(offscreenCanvas)

  // ③ 叠加 FOV 遮罩
  drawFogOverlay()

  // ④ 绘制动态层
  drawItems()
  drawEnemies()
  drawPlayer()
  drawStairs()

  // ⑤ UI
  drawMinimap()
  drawFloorIndicator()
```

**性能优化**：
- 静态地图缓存到离屏 Canvas
- 只在地城生成/切换楼层时重绘静态层
- 每帧只重绘动态层（玩家/敌人位置变化）

### 10.7 小地图

在主 Canvas 左上角绘制缩小版全览：

```
renderMinimap(dungeon):
  scale = 3px/格
  for each cell:
    if revealed && != WALL: draw filled rect (grey)
    if == stairs && revealed: draw gold rect
    if == player: draw blue rect
```

尺寸约 120×90px，离屏 Canvas 缓存。
