## 一、保留的设计（共识认可）

1. **"Boss = 规则叠加物"** — Boss 不是血条容器而是规则修改器。方向被一致认可，作为战斗系统核心骨架保留。
2. **不可逆职业分支切换** — 消耗"命运碎片"切换专精，增加决策重量感。方向正确。
3. **buff 组合系统** — buff×buff 触发隐藏效果，增加复玩价值。保留。
4. **Roguelite 元成长** — 死亡后灵魂碎片升级永久天赋树。方向正确。

## 二、高优修改（共识问题）

### 2.1 Scope 过大 — MVP 必须裁剪

**问题**：6 个专精 × 技能树 + 5 栏位装备 + buff 池 + 地城生成，内容量远超单文件 50KB 限制。

**裁决 — MVP 范围**：

| 系统 | 原计划 | MVP 裁剪后 |
|------|--------|-----------|
| 职业 | 3 基础 + 6 专精 | **1 个**（战士），无专精分支 |
| buff 选择 | 每层选 3 个 | **3 选 1** |
| 技能树 | 6 棵 × 3 支线 | **8-12 个"纹章"**，平铺无树状结构 |
| 装备 | 5 栏位 × 5 稀有度 | **3 栏位**（武器/护甲/饰品），3 稀有度 |
| 地城 | 5 层 + 隐藏房间 | **3 层**，无隐藏房间 |
| 永久天赋 | 完整树 | **5 个线性选项** |

**原则**：先跑通"探索→战斗→成长→Boss"核心循环，再扩展。

### 2.2 伤害公式需调整

**问题**：`DEF/(DEF+50)` 减伤公式在高 DEF 时收益递减过快。坦克职业 DEF 40 时减伤 44%，但 DEF 100 时减伤 66%——投入翻倍收益只增 22%。

**建议修改**：
```
减伤 = min(0.8, 目标DEF × 0.008)   // 线性减伤，上限 80%
// DEF=50 → 40% 减伤
// DEF=100 → 80% 减伤（上限）
// DEF=150 → 80% 减伤（已封顶）
```

**或保留原公式但调整系数**：
```
减伤 = 目标DEF / (目标DEF + 100)   // 分母增大，曲线更平缓
// DEF=50 → 33%
// DEF=100 → 50%
// DEF=150 → 60%
```

### 2.3 等级与地城深度不匹配

**问题**：20 级上限 + 5 层地城。如果每层打 3-5 场战斗 + 1 个 boss，5 层 ≈ 20-25 场战斗。升级太快（第 3 层就可能满级）或太慢。

**建议方案（二选一）**：

**方案 A：地城内不升级，通关后升**
- 地城内靠装备 + buff 驱动成长
- 通关后获得大量 EXP → 永久性角色变强
- 下局地城从更高等级开始
- 符合 Roguelite 节奏

**方案 B：等级 × 赛季循环**
- 每局地城前重置等级到 1
- 地城内有经验书/战斗 EXP
- 本局最高到 5-8 级
- 通关后"灵魂碎片"解锁永久天赋
- 每 5 次通关 = 一个"赛季"，解锁新内容

**推荐方案 B**——每局有成长曲线感，通关有永久成就感。

### 2.4 认知负荷过高

**问题**：等级 + 装备 + 技能树 + buff + 职业 五个系统同时存在，新手第一局无法理解关系。

**建议分层暴露**：

```
第 1 局：  等级 + 基础战斗（只有 ATK 和 HP）
第 2 局：  + 装备系统（捡到的比身上的好）
第 3 局：  + buff 选择（层前 3 选 1）
第 5 局：  + 技能树（通关解锁）
第 10 局： + 职业分支
```

**具体做法**：
- 不做大段教程文字
- 让系统自然暴露——前 2 层不出现装备掉落，第 3 层才开始
- buff 选择界面用进度性揭示（灰显未解锁的功能）

### 2.5 死亡惩罚过重

**问题**：死亡丢失本局全部装备 + 金币 + 消耗品。玩家可能花了 20 分钟积累，一次死亡全归零。

**建议修改**：
- 保留**一件**本局装备（"遗物"概念，和 buff 的"永恒遗物"呼应）
- 保留**50%** 金币
- 新增"遗物槽"——每多保存一个遗物，消耗一个槽位（稀缺资源）
- 死亡前如果遇到"灵魂商人"，可以用金币购买"安全返回"（回入口但保留一切）

## 三、争议点裁决

### 3.1 技能树：树状 vs 平铺

**正方**：树状结构有探索感，点亮终极技能的成就感强
**反方**：树状结构在单文件下 UI 复杂，平铺"纹章"系统更简洁

**裁决**：MVP 用平铺"纹章"，后续迭代升级为树状。纹章本质是技能树的扁平化——每个纹章是一张"像素卡片"，上面有图标、名字、效果描述，选择后永久装备。

### 3.2 职业：开局选 vs 后期解锁

**正方**：开局选职业，玩家有身份认同感
**反方**：开局选择增加决策压力，应该让玩家先体验再选择

**裁决**：MVP 无职业选择（默认战士），跑通后再加。原因：单文件约束下，1 个职业的内容量已经很大。

## 四、创新机会

### 4.1 "Boss = 规则叠加物"深化

每个 Boss 不是"更大血条"，而是修改游戏规则：

| Boss | 规则修改 | 对策 |
|------|----------|------|
| 暗影法师 | 每回合随机禁用一个技能 | 需要多技能组合 |
| 贪婪之王 | 战斗内装备属性逐回合衰减 | 需要爆发，不能拖 |
| 混沌之龙 | 地图每 3 回合翻转（上下颠倒） | 位置策略型 |
| 最终 Boss | 叠加以上所有规则 | 全面考验 |

**这比血条 boss 有趣 10 倍**——玩家面对的是"解谜"而非"数值碾压"。

### 4.2 地城环境 = 动态 buff

地城不只是战斗舞台，环境本身就是 buff/debuff：

- **毒雾区域**：每回合 -5% HP，但击杀敌人获得毒抗 buff
- **遗迹回廊**：战斗间隔出现回忆碎片，拼凑出完整 buff
- **岩浆层**：装备每 2 回合有概率"过热"（暂时失效）
- **星空顶**：每晚一回合，所有 buff 效果翻倍

### 4.3 战斗结果影响地城

传统地城是静态地图。改进：**战斗行为改变地城结构**。

- 战斗中使用了火焰技能 → 战后该房间着火了，封锁一条路
- 战斗中使用治疗 → 战后该房间出现花丛，隐藏宝箱
- 战斗死亡后复活 → 地城结构改变，出现"亡者之路"

### 4.4 "Build 名片"分享

单文件游戏天然适合分享。每局结束后生成一张像素风"Build 名片"：

```
┌────────────────────────────┐
│  ⚔️  狂战士 · 第 5 层       │
│  ATK:85 HP:400 DEF:35      │
│  ┌────┐ ┌────┐ ┌────┐     │
│  │⚔️  │ │🛡️  │ │🔥  │ buff │
│  │狂怒│ │铁壁│ │烈焰│     │
│  └────┘ └────┘ └────┘     │
│  纹章: 旋风斩·死亡之舞      │
│  耗时: 23 分 17 秒         │
└────────────────────────────┘
```

玩家截图分享 → 自然传播。

## 五、MVP 裁剪建议

### MVP 范围（第三版，约 2-3 周）

| 系统 | MVP 包含 | MVP 不包含 |
|------|----------|-----------|
| 战斗 | 回合制 + 技能释放 + 先手判定 | 连携技、回合外行动 |
| 成长 | 等级 + 3 选 1 buff + 3 栏位装备 | 技能树、职业分支 |
| 地城 | 3 层 × 3 房间 + boss | 隐藏房间、环境互动 |
| Boss | 2 个规则型 Boss | 多阶段、合体 |
| 永久 | 灵魂碎片 + 5 天赋 | 赛季系统、成就 |
| UI | HTML 弹窗 + Canvas 战斗 | 像素风 NPC、对话系统 |

### 开发顺序

```
1. 战斗框架（回合制、技能、先手）     ← 核心循环
2. 地城推进（3层→boss→结算）          ← 验证玩法
3. buff 框架接入                      ← 差异化
4. 装备掉落 + 背包                    ← 成长维度
5. 等级 + EXP                         ← 成长维度
6. Boss 规则系统                      ← 亮点
7. 永久天赋 + 存档                    ← 留存
8. 像素风打磨                         ← 视觉
```

## 六、更新后的成长系统概要

MVP 单职业（战士），等级 1-10，地城内靠装备 + buff 驱动成长。每层前 3 选 1 buff，buff 与装备属性联动。装备 3 栏位（武器/护甲/饰品），3 稀有度（白/蓝/紫），随机词缀。Boss 是规则修改器而非血条。通关获得灵魂碎片，解锁 5 个永久天赋。等级 × 赛季循环：每局从 1 级开始，通关永久变强。
